|-------|-------|-------|

## Color

From `src/theme/colors.ts`:

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#151718` (background) | Screen backgrounds, tab bar, FlatList background |
| Secondary (30%) | `#1E2024` (surface) + `#24272C` (surfaceElevated) | Meal cards, bottom sheet body, input row backgrounds |
| Accent (10%) | `#8DC28A` (accent) | Specific elements listed below |
| Accent dim | `#1A3326` (accentDim) | Subtle backgrounds behind accent-bordered cards only |
| Ghost/Placeholder | `#8E9298` (secondary) | Ghost text, "last: Xg" badge text, placeholder values |
| Destructive | `#D9534F` (danger) | Swipe-to-delete reveal area only |

**Accent (`#8DC28A`) is reserved for:**
1. Repeat icon on meal cards (meals with `meal_foods` data)
2. "Save to Meal Library" toggle when checked/active
3. CTA button background: "LOG MEAL", "SAVE CHANGES", "SAVE TO LIBRARY"
4. Active state of Save-to-library checkbox border + fill
5. Activity indicators and refresh control tint
6. Calories preview text in FoodGramInput (existing pattern, preserved)

**Accent is NOT used for:** general borders, card backgrounds, section headers, inactive states, the "last: Xg" badge background.

**Ghost text color:** `colors.secondary` (#8E9298) — applies to both `placeholderTextColor` on the gram TextInput (remembered portion pre-fill) and the "last: Xg" badge text on frequent food cards. Consistent with the existing `GhostReference` component pattern in `SetLoggingPanel`.

**Source:** `src/theme/colors.ts` (definitive), confirmed against `MealBuilderScreen.tsx`, `FoodGramInput.tsx`, `GhostReference.tsx`, `MealListItem.tsx`.

## Component Inventory

New and modified UI elements for Phase 40:

### 1. Repeat Icon — MealListItem (modified)

- **Visibility:** Render only when `meal.hasMealFoods === true` (new prop). Absent for manual-entry meals.
- **Icon:** Unicode repeat symbol `↺` (U+21BA) OR a simple two-arrow circular glyph rendered as a `Text` component — consistent with existing text-symbol approach. Do NOT introduce an icon library.
- **Size:** 20px font size, `colors.accent`, positioned at trailing edge of meal row, vertically centered.
- **Touch target:** Wrap in `TouchableOpacity` with `hitSlop={{ top: 12, bottom: 12, left: 12, right: 12 }}`, `minWidth: 44`, `minHeight: 44`. `activeOpacity={0.7}`.
- **Position:** Right side of row, left of the swipe-delete dead zone. Add `marginLeft: spacing.md` between repeat icon and meal content.
- **Animation on tap:** None on the icon itself. The builder opens via navigation, pre-loading provides implicit feedback.

### 2. "last: Xg" Badge — FoodSearchModal frequent food cards (modified)

- **Visibility:** Render only when `lastUsedGrams` value exists for this food. Absent for first-time foods.
- **Format:** `"last: {X}g"` — lowercase, compact. Example: `"last: 150g"`.
- **Style:** Inline badge within the frequent food card, alongside existing usage count and macro summary. No background fill — plain `Text` using `fontSize.sm` (13px), `weightRegular` (400), `colors.secondary` (#8E9298). Separated from adjacent content by `spacing.sm` (8px) gap.
- **Do NOT use:** accent color, pill/badge background shape, or bold weight for this element — it is reference data, not interactive.

### 3. Ghost Gram Pre-fill — FoodGramInput (modified)

- **Mechanism:** When `lastUsedGrams` is provided as a prop, pass it as the `placeholder` value on the gram `TextInput` (not as `value` — user must type to confirm).
- **Styling:** `placeholderTextColor={colors.secondary}` — same as all other placeholders in the app. No special ghost overlay needed.
- **Behavior:** Typing any character replaces placeholder cleanly. `selectTextOnFocus` remains true.
- **Empty field state:** When no `lastUsedGrams` exists, placeholder reverts to `"0"` (current default). No visual change to empty-state appearance.

### 4. Builder Header Mode Indicator — MealBuilderScreen (modified)

- **Edit mode header:** Text `"Edit Meal"` — `fontSize.md` (17px), `weightSemiBold` (600), `colors.primary`.
- **Normal mode header:** Text `"Build Meal"` — same style. No animation between modes.
- **Library mode header:** Text `"Build Meal"` — same style. Mode is implied by CTA label, not header.
- **Visual cue beyond header:** No color change, no border change. The CTA label is the primary mode signal.

### 5. CTA Button Variants — MealBuilderScreen (modified)

| Mode | CTA Label | Color |
|------|-----------|-------|
| Normal (log to diary) | `LOG MEAL` | `colors.accent` bg, `colors.onAccent` text |
| Edit (update existing meal) | `SAVE CHANGES` | `colors.accent` bg, `colors.onAccent` text |
| Library (save to library only) | `SAVE TO LIBRARY` | `colors.accent` bg, `colors.onAccent` text |

- All CTAs: `fontSize.base` (15px), `weightSemiBold` (600), `borderRadius: 12`, `paddingVertical: spacing.md` (12px), `minHeight: 48px` — matches existing `FoodGramInput` submit button pattern exactly.
- Disabled state: `opacity: 0.5` — matches existing `submitButtonDisabled` pattern.

### 6. Save-to-Library Toggle — MealBuilderScreen (new element)

- **Position:** In the builder's footer section, above the CTA button. Below the meal type/date selectors.
- **Style:** Row with `flexDirection: 'row'`, `alignItems: 'center'`, `paddingHorizontal: spacing.base`, `paddingVertical: spacing.md`.
- **Label text:** `"Save to Meal Library"` — `fontSize.base` (15px), `weightRegular` (400), `colors.primary`.
- **Toggle control:** Use React Native `Switch` component. `trackColor={{ false: colors.surfaceElevated, true: colors.accentDim }}`. `thumbColor={isEnabled ? colors.accent : colors.secondary}`. `ios_backgroundColor={colors.surfaceElevated}`.
- **Default state:** Unchecked (false) when opened from AddMealModal normal flow. Pre-checked (true, locked) when opened from AddLibraryMealModal (library mode — toggle disabled, grayed out to communicate the locked state: `opacity: 0.5` on the toggle control only, not the label).
- **Hidden when:** Builder is in edit mode (`mode === 'edit'`).

### 7. "Build Meal" Button — AddLibraryMealModal (new element)

- **Pattern:** Identical to the existing "Build Meal" button in `AddMealModal` — same style, same `colors.accent` background, same `colors.onAccent` text, same `borderRadius: 12`.
- **Label:** `"BUILD MEAL"` — uppercase, `fontSize.base` (15px), `weightSemiBold` (600).
- **Position:** Primary action area, alongside (not replacing) the existing manual entry flow.

### 8. Meal Card Tap Target — MacrosView / MealListItem (modified behavior)

- **No visual change** to the existing meal card appearance from the tap area expansion.
- Routing logic (builder vs modal) is invisible to the user — the navigation destination differs but the tap gesture is identical.
- The repeat icon (§1 above) is the only new visual element on the meal card.

## Interaction Patterns

### Repeated Meal Pre-load Animation

When MealBuilderScreen opens with a pre-loaded food list (repeat or edit mode), the food cards appear without entrance animation — render immediately. The existing slide-up entrance of the builder screen itself provides sufficient motion feedback. Do NOT add staggered item animations; this phase must not introduce new animation infrastructure.

### Toast / Haptic Feedback

- **Successful meal log (normal mode):** Existing toast/haptic pattern — no change.
- **Successful edit save:** Same toast/haptic as normal log. Copy: `"Meal updated"` (short, dismissive).
- **Successful library save (library mode):** Toast copy: `"Saved to library"`. Same haptic as log.
- **Successful repeat open:** No toast. Navigation is the feedback. Haptic: light impact on tap (matches existing repeat/action tap patterns in app).

### Builder Mode Detection

Via React Navigation `route.params`. Param key: `mode`. Values: `'normal'` | `'edit'` | `'library'`. Downstream components read `mode` once at mount — no reactive mode changes within a session. This is a discretion decision (CONTEXT.md §Claude's Discretion).

## Accessibility

- All new `TouchableOpacity` elements: `accessibilityRole="button"`, `accessibilityLabel` matching the visible label.
- Repeat icon: `accessibilityLabel="Repeat meal"`, `accessibilityHint="Opens builder pre-loaded with this meal's foods"`.
- Save-to-library Switch: `accessibilityLabel="Save to Meal Library"`, `accessibilityRole="switch"`.
- Ghost placeholder text: Not read as content by screen readers (placeholder is dismissed when value exists — React Native default behavior). Acceptable.
- Color is never the sole indicator: repeat icon uses a symbol character, toggle uses thumb position, CTA label text communicates mode.
- Minimum contrast: all text colors (`colors.primary` #FFFFFF and `colors.secondary` #8E9298) on `colors.surface` (#1E2024) and `colors.surfaceElevated` (#24272C) pass WCAG AA (verified: white on #1E2024 = 14.5:1; #8E9298 on #1E2024 ≈ 4.2:1 — borderline AA for 13px text; acceptable given it is reference/secondary data, not critical action text).
