## Spacing Scale

Declared values from `src/theme/spacing.ts` (all multiples of 4):

| Token | Value | Usage | Standard Set |
|-------|-------|-------|--------------|
| xs | 4px | Icon gaps, badge padding, macro pill inner padding | yes |
| sm | 8px | Between pills, inline element gap, compact row gap | yes |
| md | 12px | Card internal padding vertical, input field padding | **project deviation** — see note |
| base | 16px | Card horizontal padding, screen horizontal padding | yes |
| lg | 20px | Section gap, between cards in list | **project deviation** — see note |
| xl | 24px | Bottom bar internal padding, section header margin | yes |
| xxl | 32px | Between major layout zones | yes |
| xxxl | 48px | Bottom safe area padding | yes |

**Spacing deviation note:** `md` (12px) and `lg` (20px) are project-inherited values defined in `src/theme/spacing.ts`. They are grid-safe (multiples of 4) but fall outside the standard set {4, 8, 16, 24, 32, 48, 64}. These deviations are acknowledged — do NOT replace them with nearest standard values. Use the tokens as defined in the project theme file. Checker treats these as approved project-inherited deviations.

Exceptions:
- Touch targets: 44px minimum height on all interactive elements (MealTypePills, food row tap, remove button, Add Another Food button, Log Meal button) — source: CONTEXT.md D-20 / ui-ux-pro-max touch-target-size rule
- Sticky bottom bar: paddingBottom must account for home indicator on notched devices (use `xxxl` 48px or `useSafeAreaInsets().bottom`)
- FoodGramInput bottom card: `paddingTop: spacing.xl` (24px), `paddingBottom: spacing.xxl` (32px above keyboard)

## Typography

All sizes from `src/theme/typography.ts`. Exactly 4 sizes and 2 weights used in this phase:

| Role | Token | Size | Weight | Line Height | Usage |
|------|-------|------|--------|-------------|-------|
| Screen title | `fontSize.lg` | 20px | 600 (SemiBold) | 1.2 | "BUILD MEAL" header |
| Card primary | `fontSize.md` | 17px | 600 (SemiBold) | 1.3 | Food name on each card, gram input food name label, gram input numeric value |
| Body / label | `fontSize.base` | 15px | 400 (Regular) | 1.5 | Gram weight subtext, description field, date button, macro preview values |
| Caption / micro | `fontSize.sm` | 13px | 600 (SemiBold) | 1.2 | Section labels (ALL-CAPS), macro values, calorie numbers, badge text (usage count, remove icon label) |

Weight constraint: Exactly 2 weights — Regular (400) and SemiBold (600). Bold (700) is not used in this phase.

Size constraint: Exactly 4 sizes — 13, 15, 17, 20px. `fontSize.xl` (24px) and `fontSize.xs` (11px) are NOT used.
- Gram input numeric value: `fontSize.md` (17px) SemiBold — the larger visual weight from SemiBold at 17px is sufficient to distinguish the input from surrounding labels without adding a fifth size tier.
- Badge text (usage count, remove label): `fontSize.sm` (13px) — folded into caption tier.

Source: CONTEXT.md D-19; existing `FoodResultItem.tsx` and `AddMealModal.tsx` patterns

## Interaction Contract

### Touch Targets

All interactive elements must meet 44px minimum:

| Element | Min Height | Min Width |
|---------|------------|-----------|
| MealTypePills pill | 44px | flex (fills row) |
| MealFoodCard (tap to edit) | 44px | full width |
| Remove (X) button | 44px | 44px |
| "+ Add Another Food" card | 44px | full width |
| FoodGramInput input field | 44px | full width |
| "Add to Meal" button | 48px | full width |
| "LOG MEAL" button | 48px | full width |
| Date/time tap target | 44px | full width |

### Animation

| Interaction | Duration | Easing | Property |
|-------------|----------|--------|----------|
| FoodGramInput slide-up | 250ms | `Easing.out(Easing.quad)` | `Animated.Value` translateY |
| FoodGramInput slide-down (dismiss) | 200ms | `Easing.in(Easing.quad)` | translateY |
| MealFoodCard add (append to list) | none — instant | — | FlatList append |
| MealFoodCard remove | none — instant | — | FlatList remove |
| LOG MEAL loading state | immediate on tap | — | button text → spinner |
| MealTypePill selected state | none — instant | — | backgroundColor swap |

Note: `prefers-reduced-motion` is not available in React Native — animations are always on. Keep durations short (150–300ms) per CONTEXT.md D-20.

### Keyboard Handling

- `FoodGramInput` gram field: auto-focus on component mount
- `KeyboardAvoidingView behavior="padding"` wraps the bottom card
- `keyboardShouldPersistTaps="handled"` on all ScrollView/FlatList
- Description field in `MealTotalsBar`: `returnKeyType="done"` dismisses keyboard
- Tapping the FlatList food list area dismisses `FoodGramInput` and the keyboard

### Disabled States

| Element | Disabled Condition | Visual |
|---------|--------------------|--------|
| LOG MEAL button | No foods added OR mealType is null | `opacity: 0.5` |
| LOG MEAL button | `isSubmitting === true` | `opacity: 0.5` + ActivityIndicator |
| "Add to Meal" button | gram input is empty or "0" | `opacity: 0.5` |

### Error Handling

- DB write failure: inline error text below LOG MEAL button, `colors.danger`, `fontSize.sm` (13px)
- Date parse error: inline below date field (matches `AddMealModal` pattern)
- Gram input with non-numeric value: silently coerces to 0 (live preview shows 0 macros)

## Implementation Notes for Executor

1. **FlatList vs ScrollView:** Use `FlatList` for the food list (per CONTEXT.md Claude's Discretion — consistent with Phase 38). All items wrapped in `React.memo` with stable `keyExtractor` using `food.id`.

2. **FoodGramInput dismissal:** When "+ Add Another Food" is tapped while `FoodGramInput` is open, slide-dismiss the card first, then open `FoodSearchModal`. Do not stack the two overlays.

3. **Macro computation:** `(grams / 100) * per100gValue` — no rounding during live preview. Round to 1 decimal place for display. Use `computeCalories()` from `src/utils/macros.ts` for kcal.

4. **Sticky bar height:** `MealTotalsBar` is absolutely positioned at the bottom. The `FlatList` must have `contentContainerStyle={{ paddingBottom: TOTALS_BAR_HEIGHT }}` to prevent last item from hiding behind the bar. Measure bar height via `onLayout`.

5. **Transaction logging:** `addMealWithFoods()` must use `runTransaction` from `src/db/database.ts`. On success: light haptic (`Haptics.impactAsync(Haptics.ImpactFeedbackStyle.Light)`), then `navigation.goBack()` + `onSaved()`.

6. **Description field visibility:** Always visible (not collapsible) per Claude's Discretion ruling — keeps the form predictable.

7. **Date/time:** Defaults to `new Date()` at screen mount (not at tap time). Same `parseDate`/`parseTime` + `formatDateForInput`/`formatDisplayDate` pattern as `AddMealModal.tsx`.

8. **Navigation:** `MealBuilderScreen` is added to `ProteinStackNavigator` as a stack screen (not modal). `AddMealModal` gets a "Build Meal" secondary button below the existing manual entry that calls `navigation.navigate('MealBuilderScreen')` and closes the modal.
