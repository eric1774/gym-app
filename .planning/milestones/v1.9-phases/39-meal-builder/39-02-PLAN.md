Props interface:
```typescript
interface FoodGramInputProps {
  food: {
    name: string;
    proteinPer100g: number;
    carbsPer100g: number;
    fatPer100g: number;
  };
  initialGrams?: string;
  onSubmit: (grams: number) => void;
  onDismiss: () => void;
  visible: boolean;
  buttonLabel?: string; // "Add to Meal" (default) or "Update" for edit
}
```

**Animation (per UI-SPEC):**
- Use `Animated.Value` for translateY
- Slide-up: 250ms, `Easing.out(Easing.quad)` when `visible` becomes true
- Slide-down: 200ms, `Easing.in(Easing.quad)` when dismissed
- Use `useEffect` watching `visible` prop to trigger animation
- Initial translateY = 400 (off-screen below), animate to 0 when visible

**Layout structure:**
- Outer: semi-transparent backdrop `<Pressable>` with `backgroundColor: 'rgba(0,0,0,0.4)'`, `position: 'absolute'`, covering full screen. `onPress={onDismiss}` to dismiss on backdrop tap.
- Inner: `<Animated.View>` positioned at bottom, `backgroundColor: colors.surface`, `borderTopLeftRadius: 20`, `borderTopRightRadius: 20`, `paddingTop: spacing.xl` (24px), `paddingBottom: spacing.xxl` (32px), `paddingHorizontal: spacing.base` (16px)
- Wrap inner content in `<KeyboardAvoidingView behavior="padding">` for keyboard handling

**Content rows:**
1. Food name: `<Text>` in `fontSize.md` (17px), `weightSemiBold`, `colors.primary`
2. Gram input row: `<TextInput>` with `keyboardType="numeric"`, auto-focused on mount via `useEffect` with `ref.current?.focus()` and 100ms delay (per Android keyboard timing pattern from Phase 38). `fontSize.md` (17px), `weightSemiBold`, `colors.primary` text. Background `colors.surfaceElevated`, `borderRadius: 8`, `padding: spacing.md`, `minHeight: 44`. Show "g" suffix label to the right of input.
3. Live macro preview: horizontal row showing computed P, C, F values and calories. Computed as `(parseFloat(gramsText) || 0) / 100 * per100gValue` for each macro. Display with 1 decimal place rounding: `value.toFixed(1)`. Use MACRO_COLORS for each macro label. Show `computeCalories(p, c, f)` rounded to nearest integer for kcal. `fontSize.base` (15px). Update on every keystroke (no debounce — per D-08, pure computation).
4. Submit button: `<TouchableOpacity>` with `backgroundColor: colors.accent`, `borderRadius: 12`, `paddingVertical: spacing.md` (12px), `minHeight: 48`, `alignItems: 'center'`. Text: `buttonLabel ?? 'Add to Meal'` in `colors.onAccent`, `fontSize.base`, `weightSemiBold`. Disabled with `opacity: 0.5` when grams is empty or "0" (per UI-SPEC disabled states).

**State management:**
- `const [gramsText, setGramsText] = useState(initialGrams ?? '')` — string state for text input
- Compute macros on every render (pure math, no useEffect needed):
  ```typescript
  const grams = parseFloat(gramsText) || 0;
  const protein = (grams / 100) * food.proteinPer100g;
  const carbs = (grams / 100) * food.carbsPer100g;
  const fat = (grams / 100) * food.fatPer100g;
  const calories = Math.round(computeCalories(protein, carbs, fat));
  ```
- Submit handler: `onSubmit(grams)` — only called when grams > 0

**When `visible` is false:** Return null (component unmounts, animation handles exit).

Actually, for smooth exit animation: keep mounted but animate out, then call `onDismiss` after animation completes. Use a local `isAnimating` state or just let the parent control `visible`.

Better approach: Always render when `visible || isAnimatingOut`. Use a ref to track animation state. When `visible` goes false, animate out, then set internal `shouldRender = false`.

Export as `export function FoodGramInput({ ... }: FoodGramInputProps)`.
