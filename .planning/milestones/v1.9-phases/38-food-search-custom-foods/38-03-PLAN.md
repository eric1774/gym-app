State:
- `name: string` — initialized from `initialName` (per D-10: search query pre-fills name)
- `proteinStr: string` — empty string initially
- `carbsStr: string` — empty string initially
- `fatStr: string` — empty string initially
- `isSaving: boolean` — false initially
- `errors: Record<string, string | null>` — `{ name: null, protein: null, carbs: null, fat: null }`

Validation (called on save):
- `name.trim().length === 0` => set `errors.name = 'Required'`
- For each macro field: if string is empty => 'Required'. If `isNaN(parseFloat(value))` => 'Enter a number'. If `parseFloat(value) < 0` => 'Enter a number'.
- Return true only if all errors are null.

Auto-computed calories:
- `const parsedProtein = parseFloat(proteinStr) || 0;`
- `const parsedCarbs = parseFloat(carbsStr) || 0;`
- `const parsedFat = parseFloat(fatStr) || 0;`
- `const caloriePreview = Math.round(computeCalories(parsedProtein, parsedCarbs, parsedFat));`
- Display as read-only field (per D-11)

Save handler:
1. Run validation. If invalid, return.
2. Set `isSaving = true`.
3. Call `await foodsDb.createCustomFood(name.trim(), parsedProtein, parsedCarbs, parsedFat)`.
4. `Vibration.vibrate(50)` for success haptic (per D-20).
5. Call `onFoodCreated(createdFood)` — the parent (FoodSearchModal) will call `onFoodSelected` and close the modal (per D-13: auto-select + close).
6. If error: set a generic error message, set `isSaving = false`.
