## Component Inventory

### FoodSearchModal

- Full-screen modal, slides up from bottom using `Modal` with `animationType="slide"`
- Background: `colors.background` (#151718)
- Header bar: modal title "Search Foods" at `fontSize.md` / `weightSemiBold` / `colors.primary`, left-side back/close chevron at 44px touch target
- Animation: `animationType="slide"` (React Native Modal built-in), 300ms (platform default)
- Auto-focus search bar on mount via `TextInput` `autoFocus` prop
- Keyboard: `KeyboardAvoidingView` wrapping entire modal content (same pattern as `AddMealModal.tsx`)
- Dismiss keyboard on scroll: `FlatList` `keyboardShouldPersistTaps="handled"`

### SearchBar

- Background: `colors.surface` (#1E2024), `borderRadius: 10` (CONTEXT.md D-04)
- Height: 48px
- Placeholder: "Search foods..." in `colors.secondary`
- Font: `fontSize.base` (15px), `weightRegular`
- Horizontal padding: `spacing.base` (16px)
- Debounce: 200ms (CONTEXT.md D-02)
- Clear button: appears when text.length > 0, tapping clears input and resets to frequent foods view

### FoodResultItem (search result card)

- Background: `colors.surfaceElevated` (#24272C), `borderRadius: 14` (CONTEXT.md D-19)
- Padding: `spacing.base` (16px) horizontal, `spacing.md` (12px) vertical
- Min-height: 44px touch target
- Layout (vertical stack):
  - Line 1: Food name — `fontSize.md` (17px), `weightSemiBold`, `colors.primary`
  - Line 2: Category — `fontSize.base` (15px), `weightRegular`, `colors.secondary`
  - Line 3: Macro summary — `fontSize.sm` (13px), `weightRegular`, `colors.secondary` — format: `P:Xg  C:Xg  F:Xg  |  Xkcal  per 100g`
- Spacing between lines: `spacing.xs` (4px)
- Card gap in list: `spacing.sm` (8px) via `FlatList` `ItemSeparatorComponent`
- Memoized: `React.memo` on component (CONTEXT.md D-18, mobile-design skill)
- `onPress`: triggers `onFoodSelected` callback with food object, closes modal

### FrequentFoodsSection

- Section header: "FREQUENTLY LOGGED" — `fontSize.xs` (11px), `weightSemiBold`, `colors.secondary`, letter-spacing: 0.8
- Shown before any text is typed (pre-search state)
- Up to 10 items, same `FoodResultItem` card layout PLUS usage count badge
- Usage count badge: pill shape, `colors.accentDim` (#1A3326) background, `colors.accent` text, format "12x", `fontSize.xs`, `weightSemiBold` — positioned top-right inside card
- Empty state (no meals logged yet): single inline hint text "Your frequent foods will appear here" — `fontSize.base`, `colors.secondary`, centered, `spacing.xl` vertical padding

### NoResultsCard

- Shown when search yields 0 results
- Background: `colors.surface`, `borderRadius: 14`, `spacing.base` padding
- Text line 1: `No results for "[query]"` — `fontSize.md`, `weightSemiBold`, `colors.primary`
- Text line 2: `Try a different spelling or create a custom food` — `fontSize.base`, `weightRegular`, `colors.secondary`
- CTA button below text: "+ Create Custom Food" — `colors.accent` label, `fontSize.base`, `weightSemiBold`, min-height 44px
- Gap between text and CTA: `spacing.md` (12px)

### CustomFoodForm (inline within FoodSearchModal)

- Slides in over the search view via state transition (not a separate modal)
- Back arrow (←) header at top: `colors.primary`, 44px touch target, navigates back to search view
- Header title: "New Custom Food" — `fontSize.md`, `weightSemiBold`, `colors.primary`
- Search query pre-filled into Name field on open (CONTEXT.md D-10)
- Fields (vertical stack, `spacing.base` gap between fields):
  - Name: text input, required, `fontSize.base`, `colors.primary`
  - Protein per 100g: numeric input, required, suffix label "g", `colors.primary`
  - Carbs per 100g: numeric input, required, suffix label "g", `colors.primary`
  - Fat per 100g: numeric input, required, suffix label "g", `colors.primary`
  - Calories (read-only): computed via `computeCalories()`, shown as `fontSize.base`, `colors.secondary`, italic or muted — updates live as macros are entered
- Field labels: `fontSize.sm` (13px), `weightSemiBold`, `colors.secondary`, above each input
- Field background: `colors.surface`, `borderRadius: 10`, height 48px, `spacing.base` horizontal padding
- Save CTA: "Save Custom Food" — full-width, `colors.accent` background, `colors.onAccent` (#1A1A1A) text, `fontSize.base`, `weightSemiBold`, height 52px, `borderRadius: 14`
- Save behavior: inserts food, triggers haptic feedback (success pattern), auto-selects new food via `onFoodSelected`, closes modal

## Interaction Contract

| Trigger | Response | Timing |
|---------|----------|--------|
| Modal opens | Search bar auto-focused, keyboard appears | Immediate (autoFocus) |
| User types | Results update via debounced SQLite query | 200ms debounce |
| No results | NoResultsCard with "+ Create Custom Food" CTA | Immediate on empty result |
| Tap "Create Custom Food" | CustomFoodForm slides into view | No animation needed — state swap |
| Tap food result | `onFoodSelected` fires, modal closes | Haptic (light impact), immediate close |
| Tap back arrow in CustomFoodForm | Returns to search view with prior query intact | Immediate |
| Save custom food | Insert DB, haptic success, auto-select, modal closes | Haptic + close |
| Scroll results | Keyboard dismissed (`keyboardShouldPersistTaps="handled"`) | Immediate |

Haptics: React Native `Vibration` or `react-native-haptic-feedback` — existing pattern in codebase. Light impact on food selection; success pattern on custom food save.

Source: CONTEXT.md D-20 ("haptic feedback on food selection"), mobile-design skill (touch-psychology.md).
