Props interface:
```typescript
interface FoodSearchModalProps {
  visible: boolean;
  onClose: () => void;
  onFoodSelected: (food: Food) => void;
}
```

**State management:**
- `query: string` — search input text
- `results: FoodSearchResult[]` — search results from foodsDb.searchFoods
- `frequentFoods: FoodSearchResult[]` — from foodsDb.getFrequentFoods
- `frequentLoading: boolean` — loading state for initial frequent foods fetch
- `showCustomForm: boolean` — whether custom food form is visible (will be used by Plan 03; initialize as `false`, not rendered yet)
- `searchInputRef: React.RefObject<TextInput>` — for auto-focus

**On modal open (visible becomes true):**
- Load frequent foods via `foodsDb.getFrequentFoods()` into state
- Reset query to '', results to [], showCustomForm to false

**Debounced search (200ms per D-02):**
- Use `useRef` for a timeout ID. On every query change, clear previous timeout and set a new 200ms timeout.
- When timeout fires: if query is non-empty, call `foodsDb.searchFoods(query)` and set results. If query is empty, set results to [].
- Clean up timeout on unmount.

**Food selection handler:**
- When user taps a food: `Vibration.vibrate(10)` for light haptic (per D-20), call `onFoodSelected(food)`, then `onClose()` (per D-16).

**Layout (full-screen modal per D-01):**
