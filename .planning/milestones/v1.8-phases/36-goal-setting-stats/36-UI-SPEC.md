## Spacing Scale

Declared values from `src/theme/spacing.ts` (project-specific — uses 12 as `md`, not the conventional 16):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, error text margin-top |
| sm | 8px | Compact row padding, gap between quick-add buttons, button-row gap |
| md | 12px | Card internal padding vertical (e.g., save/cancel button paddingVertical) |
| base | 16px | Default horizontal padding (paddingHorizontal on all cards and sections) |
| lg | 20px | Section spacing (e.g., marginTop below cup) |
| xl | 24px | Larger section gaps (e.g., gap between stat cards row and Log Water button) |
| xxl | 32px | Major layout gaps where needed |
| xxxl | 48px | Bottom scroll padding |

Exceptions:
- Minimum touch target: 48px height (minHeight: 48) on all interactive buttons — matches quick-add button and Log Water button established in Phase 35
- Stat cards: minHeight 80px to give adequate room for large number + label text
- Inline edit input: paddingVertical uses `sm` (8px), paddingHorizontal uses `base` (16px) — mirrors MacroProgressCard editInput
- **Non-standard multiples:** 12px (`md`) and 20px (`lg`) are not in the standard 8-point set {4, 8, 16, 24, 32, 48, 64}. Both values are project-established tokens from `src/theme/spacing.ts` and appear throughout all existing components. Their use here is justified as a project-wide consistency requirement, not a phase-specific deviation — removing them would diverge from the established codebase token system.

Source: `src/theme/spacing.ts` (codebase), `src/components/HydrationView.tsx` (minHeight: 48 on buttons), `src/components/MacroProgressCard.tsx` (editInput padding pattern).

## Typography

All values from `src/theme/typography.ts`.

Active sizes in this phase: 4 (11px, 13px, 15px, 24px).

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Section header | 13px (fontSize.sm) | 700 (weightBold) | 1.2 | "QUICK ADD", "SET YOUR DAILY WATER GOAL" card title |
| Body / label | 15px (fontSize.base) | 400 (weightRegular) | 1.5 | Goal label ("GOAL: X fl oz"), input text, error messages, stat card sublabel, stat card emoji labels |
| Stat value | 24px (fontSize.xl) | 700 (weightBold) | 1.0 | Streak count (e.g., "12"), weekly average percentage (e.g., "72%") |
| Button text | 13px (fontSize.sm) | 700 (weightBold) | 1.0 | "Set Goal", "Save Goal", "Cancel" button labels |
| Stat sublabel | 11px (fontSize.xs) | 400 (weightRegular) | 1.0 | "day streak", "7-day avg" sublabels inside stat cards |

Letter spacing: 1.2 applied to uppercase section header labels only (matches `sectionHeader` in HydrationView and `headerLabel` in MacroProgressCard).

Note: `fontSize.lg` (20px) is NOT used in this phase. Emoji labels in stat cards (fire "🔥", droplet "💧") render at `fontSize.base` (15px) — they are decorative text nodes and do not warrant a distinct size.

Source: `src/theme/typography.ts` (codebase), `src/components/MacroProgressCard.tsx` (patterns).

## Color

All values from `src/theme/colors.ts`.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#151718` (`colors.background`) | Screen background, HydrationView container |
| Secondary (30%) | `#24272C` (`colors.surfaceElevated`) | GoalSetupCard background, stat card backgrounds, inline edit TextInput background, quick-add button background |
| Accent (10%) | `#8DC28A` (`colors.accent`) | "Set Goal" button fill, "Save Goal" button fill, active underline (inherited from TabBar), streak fire emoji label color |
| Accent dim | `#1A3326` (`colors.accentDim`) | Setup card subtle background highlight (optional, use only if card needs visual separation from background) |
| Accent on | `#1A1A1A` (`colors.onAccent`) | Text on mint-filled buttons ("Set Goal", "Save Goal") |
| Secondary text | `#8E9298` (`colors.secondary`) | Stat card sublabels ("7-day avg", "day streak"), goal label default state, Cancel button text, "fl oz" suffix label |
| Primary text | `#FFFFFF` (`colors.primary`) | Stat card value numbers, TextInput entered text, GoalSetupCard title |
| Border | `rgba(255,255,255,0.05)` (`colors.border`) | Stat card border (borderWidth: 1), GoalSetupCard border, inline edit TextInput border (default state) |

Accent reserved for:
- "Set Goal" primary button background (GoalSetupCard)
- "Save Goal" primary button background (inline edit mode)
- TabBar active underline (inherited, no change needed)
- Streak stat card fire emoji color (via `colors.accent`)

The "Cancel" button uses `colors.surfaceElevated` background with `colors.secondary` text — never accent.

No destructive actions exist in this phase. `colors.danger` is used only for validation error copy (non-numeric or zero input).

Source: `src/theme/colors.ts` (codebase), 36-CONTEXT.md D-14 (stat card uses surfaceElevated + border), MacroProgressCard (editInput border, save/discard button colors).

## Component Inventory

New components to build:

### GoalSetupCard
- **File:** `src/components/GoalSetupCard.tsx`
- **Dimensions:** Full-width, paddingHorizontal from parent (`spacing.base`), internal padding `spacing.xl` (24px) vertical
- **Background:** `colors.surfaceElevated`, borderRadius 14, borderWidth 1, borderColor `colors.border`
- **Contents (top to bottom):**
  1. Title: "Set Your Daily Water Goal" — `fontSize.sm` (13px), `weightBold`, `colors.secondary`, letterSpacing 1.2, uppercase, centered
  2. Gap: `spacing.md` (12px)
  3. Input row: TextInput + "fl oz" Text side-by-side — TextInput pre-filled "64", `keyboardType="number-pad"`, `maxLength={4}`, autoFocus, `fontSize.xl` (24px), `weightBold`, `colors.primary`, centered; "fl oz" suffix at `fontSize.base` (15px), `weightRegular`, `colors.secondary`
  4. Gap: `spacing.xs` (4px) — validation error Text appears here if invalid (hidden when no error)
  5. Gap: `spacing.md` (12px)

### HydrationStatCards

- **Left card — Streak:**
  - Icon label: fire emoji ("🔥") at `fontSize.base` (15px) — displayed as Text node, decorative
  - Value: streak count integer as string — `fontSize.xl` (24px), `weightBold`, `colors.primary`
  - Sublabel: "day streak" — `fontSize.xs` (11px), `weightRegular`, `colors.secondary`
  - Empty state: value shows "0", sublabel unchanged
- **Right card — Weekly Average:**
  - Icon label: droplet emoji ("💧") at `fontSize.base` (15px) — displayed as Text node, decorative
  - Value: percentage string (e.g., "72%") — `fontSize.xl` (24px), `weightBold`, `colors.primary`
  - Sublabel: "7-day avg" — `fontSize.xs` (11px), `weightRegular`, `colors.secondary`

- **Accessibility:** `accessibilityLabel="Streak: X days"` and `accessibilityLabel="Weekly average: X%"` on each card container

## Layout & Interaction Contracts

### Conditional Rendering Gate (at HydrationView level)
```
goalOz === null
  → render <GoalSetupCard /> only
  → hide: WaterCup, goal label row, HydrationStatCards, quick-add section, Log Water button
goalOz !== null
  → render full hydration content (all elements visible)
```
Source: 36-CONTEXT.md D-01, D-04

### Full HydrationView element order (goalOz !== null, top to bottom):
1. WaterCup hero — centered, `marginTop: spacing.xl` (24px)
2. Goal label row (display or inline edit) — `marginTop: spacing.md` (12px) below cup
3. HydrationStatCards row — `marginTop: spacing.md` (12px) below goal label
4. QUICK ADD section header + button row — `marginTop: spacing.xl` (24px) below stat cards
5. Log Water button — `marginTop: spacing.xl` (24px) below quick-add row

Source: 36-CONTEXT.md D-13

### Touch Interactions
- Goal label tap → swap to inline edit mode (no animation, instant — consistent with D-03 no-animation policy)
- "Save Goal" in inline edit → validate → call hydrationDb.setWaterGoal → refreshData → swap back to display mode
- "Cancel" → swap back to display mode, no DB call, no keyboard dismiss animation
- "Set Goal" in GoalSetupCard → validate → call hydrationDb.setWaterGoal → refreshData triggers goalOz !== null → full hydration view appears

### Keyboard handling
- TextInput in GoalSetupCard: autoFocus on render. On submit (return key): trigger save. `returnKeyType="done"`.
- TextInput in inline goal edit: autoFocus on edit mode entry. On submit: trigger save. `returnKeyType="done"`.
- No custom keyboard dismiss — React Native default dismiss on outside tap applies.

### Haptics
- No haptic on goal save (goal setting is deliberate, low-frequency — haptic reserved for quick-add logging)
- Quick-add buttons retain `impactMedium` haptic from Phase 35 (no change)
