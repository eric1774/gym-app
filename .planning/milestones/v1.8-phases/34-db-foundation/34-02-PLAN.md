```typescript
export function rowToWaterLog(row: {
  id: number;
  amount_oz: number;
  logged_at: string;
  local_date: string;
  created_at: string;
}): WaterLog {
  return {
    id: row.id,
    amountOz: row.amount_oz,
    loggedAt: row.logged_at,
    localDate: row.local_date,
    createdAt: row.created_at,
  };
}

export function rowToWaterSettings(row: {
  id: number;
  goal_oz: number | null;
  created_at: string;
  updated_at: string;
}): WaterSettings {
  return {
    id: row.id,
    goalOz: row.goal_oz,
    createdAt: row.created_at,
    updatedAt: row.updated_at,
  };
}
```

**getWaterGoal()**: `SELECT * FROM water_settings LIMIT 1` — return rowToWaterSettings or null if no row.

**setWaterGoal(goalOz: number)**: Upsert pattern (same as macros setMacroGoals). COUNT, then INSERT or UPDATE goal_oz and updated_at. Return WaterSettings.

**logWater(amountOz: number, loggedAt?: Date)**: INSERT into water_logs. Use getLocalDateString for local_date, getLocalDateTimeString for logged_at and created_at. SELECT back and return WaterLog. No goal check — water logging does not require a goal to be set (unlike macros addMeal).

**getTodayWaterTotal()**: `SELECT COALESCE(SUM(amount_oz), 0) as total FROM water_logs WHERE local_date = ?` with today's local date string. Return the number.

**getStreakDays()**: Same backward-iteration algorithm as macros.ts getStreakDays, but:
- Reads `goal_oz` from `water_settings` (not `protein_goal` from `macro_settings`)
- Queries `SELECT local_date, SUM(amount_oz) as total FROM water_logs WHERE local_date <= ? GROUP BY local_date ORDER BY local_date DESC`
- Same gap-detection logic: if rowDate !== expectedDateStr, streak breaks

**get7DayAverage()**: Key difference from macros (per D-10): divide total by 7, not by active days.
```sql
SELECT COALESCE(SUM(amount_oz), 0) as total
FROM water_logs
WHERE local_date >= ? AND local_date <= ?
```
Then: `total / 7`. If total is 0 and no rows exist in range, return null. Otherwise return `Math.round(total / 7)`.

Implementation detail for get7DayAverage:
- Calculate start date as today minus 6 days (7 days inclusive of today)
- Query total oz for the range
- Also check if any rows exist in that range (to distinguish "0 oz logged" from "no data")
- If no rows exist, return null
- If rows exist, return Math.round(totalOz / 7)

A single query approach:
```sql
SELECT COUNT(*) as day_count, COALESCE(SUM(total_oz), 0) as total
FROM (
  SELECT SUM(amount_oz) as total_oz
  FROM water_logs
  WHERE local_date >= ? AND local_date <= ?
  GROUP BY local_date
)
```
If day_count === 0, return null. Otherwise return Math.round(total / 7).
