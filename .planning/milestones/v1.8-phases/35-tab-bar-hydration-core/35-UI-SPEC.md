Accent reserved for (exact list — these are the ONLY elements that use `colors.accent`):
1. Active tab underline (2px bar beneath active tab label)
2. WaterCup gradient fill (mint fill that rises proportionally from bottom)
3. "Goal met!" indicator icon/badge (shown when total >= goal)
4. "+ Log Water" full-width button background

Quick-add buttons (+8 oz, +16 oz, +24 oz) use mint text (`colors.accent`) on surface-colored background (`colors.surfaceElevated`) — NOT a solid accent fill. This matches D-11.

New components to build for this phase:

| Component | File | Size Estimate | Pattern Reference |
|-----------|------|---------------|-------------------|

| LogWaterModal | `src/screens/LogWaterModal.tsx` | ~100 lines | AddMealModal.tsx (KeyboardAvoidingView + numeric input) |

TabBar visual contract:
- Horizontal row, full-width, positioned below screen header title
- Two items: "Macros" (left) and "Hydration" (right), equal flex
- Inactive tab: label in `colors.secondary`, `fontSize.sm`, `weightBold`
- Active tab: label in `colors.primary`, `fontSize.sm`, `weightBold`, 2px `colors.accent` underline
- No background fill on tab bar, no heavy border — underline only (D-01)
- No transition animation on switch — instant content swap (D-04)
- Tab item height: 44px minimum (touch target)
- Border bottom on tab bar container: 1px `colors.border` to separate from content

WaterCup visual contract:
- Dimensions: 120px wide, 180–200px tall, centered horizontally
- Shape: rounded rectangle with `borderRadius: 16` on all corners (glass shape)
- Background (empty area): `colors.accentDim` (`#1A3326`)
- Fill (water): positioned absolutely from bottom, height = `(currentTotal / goalOz) * cupHeight`, capped at 100%
- Fill color: solid `colors.accent` (`#8DC28A`) with a subtle opacity gradient at the top edge of the water line (top 8px at 0.6 opacity, rest at 1.0) — creates soft water-surface illusion
- Cup outline: 1px `colors.border`
- Goal met state: cup fill height = 100%, overlay a checkmark icon in `colors.onAccent` centered in the cup (D-08)
- Data labels: positioned to the RIGHT of the cup (D-07)
  - Primary: "{currentTotal} / {goalOz} oz" — `fontSize.lg`, `weightBold`, `colors.primary`
  - Secondary: "{percentage}%" — `fontSize.sm`, `weightRegular`, `colors.secondary`

Quick-add buttons contract (+8 oz, +16 oz, +24 oz):
- Layout: horizontal row, `flexDirection: 'row'`, `gap: spacing.sm`, full-width
- Each button: `flex: 1`, `backgroundColor: colors.surfaceElevated`, `borderRadius: 10`, `borderWidth: 1`, `borderColor: colors.border`
- Button height: 48px minimum
- Label: "+X oz" in `colors.accent`, `fontSize.sm`, `weightBold`, centered
- Haptic feedback: `HapticFeedback.trigger('impactMedium')` on press (consistent with existing set logging haptics — D per specifics)
- No toast/confirmation on quick-add — cup fills silently (clean, instant feedback through the visual)

"+  Log Water" button contract:
- Full-width, `backgroundColor: colors.accent`, `borderRadius: 12`
- Height: 48px
- Label: "+ Log Water" in `colors.onAccent`, `fontSize.base`, `weightBold`
- Positioned below the quick-add row with `marginTop: spacing.xl`
- Matches "Meal Library" button pattern on Macros tab (D-12)

Source: D-05 through D-16 from 35-CONTEXT.md; existing component patterns.

## Interaction Contract

### Tab switching
- Tap "Hydration" tab: immediately renders HydrationView, unmounts MacrosView (D-17)
- Tap "Macros" tab: immediately renders MacrosView, unmounts HydrationView
- No swipe gesture between tabs (state-based switching per STATE.md accumulated decisions)
- Active underline moves to tapped tab — no animation, instant

### Quick-add flow
1. User taps "+16 oz" (or +8 or +24)
2. Haptic feedback fires immediately (`impactMedium`)
3. `hydrationDb.logWater(oz)` called
4. Cup fill re-renders with updated total — no modal, no toast
5. Running total label ("48 / 64 oz") updates inline
6. If total >= goal after this addition: "Goal met!" indicator appears

### Log Water modal flow
1. User taps "+ Log Water"
2. Modal slides up (React Native Modal, `animationType: 'slide'`)
3. `KeyboardAvoidingView` wraps content (behavior: `'padding'` on iOS, `'height'` on Android)
4. Numeric keyboard shown immediately (`keyboardType: 'decimal-pad'`, `autoFocus: true`)
5. User enters amount
6. Taps "Log Water":
   - Validates: amount > 0 and is a valid number
   - On invalid: shows inline error below input ("Please enter a valid amount"), does NOT close modal
   - On valid: closes modal, calls `hydrationDb.logWater(parsedOz)`, cup re-renders
7. Taps "Discard": closes modal with no changes
8. Backdrop tap: closes modal (same as Discard)

### MacrosView extraction
- ProteinScreen.tsx reduced to: header + TabBar + conditional `{activeTab === 'macros' ? <MacrosView /> : <HydrationView />}`
- All existing macros state, callbacks, and UI move verbatim into MacrosView — no behavior change (D-14, TAB-02)
- FAB remains inside MacrosView, invisible when Hydration tab is active (D-16)

Source: 35-CONTEXT.md decisions D-04, D-13, D-14, D-16, D-17; REQUIREMENTS.md HYD-02, HYD-03.
