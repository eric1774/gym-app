## Spacing Scale

Project uses named tokens from `src/theme/spacing.ts`. Values are all multiples of 4.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, tight inline padding |
| sm | 8px | Compact element spacing, margin between label and value |
| md | 12px | Card internal section gaps (project uses `md` = 12, not 16) |
| base | 16px | Standard card padding, default element spacing |
| lg | 20px | Horizontal screen padding |
| xl | 24px | Section-level vertical spacing |
| xxl | 32px | Layout gaps, major section breaks |
| xxxl | 48px | Page-level empty state padding |

Exceptions:
- Touch targets: minimum 44x44px enforced on all tappable elements (see `backButton` minWidth/minHeight in CalendarDayDetailScreen). Not applicable to Phase 26 stat rows (display only, not interactive).
- `hairlineWidth` used for stat row bottom borders in WorkoutSummary (follows existing `statRow.borderBottomWidth` pattern).

Source: `src/theme/spacing.ts` and existing screen StyleSheets.

| Role | Value | Usage |

Accent reserved for: ActivityIndicator color in loading states (existing pattern). HR stat rows use `colors.primary` for values and `colors.secondary` for labels — same as all other stat rows. Accent is NOT applied to HR stat numbers.

Secondary color detail:
- `colors.secondary` (`#8E9298`) — stat label text color ("Avg HR", "Peak HR")
- `colors.primary` (`#FFFFFF`) — stat value text color (BPM numbers)
- `colors.border` (`rgba(255,255,255,0.05)`) — hairline dividers between stat rows

Conditional visibility rule (from D-08): when `avg_hr` is null, HR stat rows are entirely absent from both views. No placeholder, no "--" display, no greyed-out row. The HR rows only render when data exists.

## Component Contracts

### WorkoutSummary — HR Stat Rows

Two new rows appended to the existing `statsContainer` in `WorkoutSummary`, rendered only when `avgHr !== null`:

```
| Avg HR    |  [N] bpm |
| Peak HR   |  [N] bpm |
```

- Layout: `flexDirection: 'row'`, `justifyContent: 'space-between'`, `alignItems: 'center'`
- Padding: `paddingVertical: spacing.md` (12px) — matches existing `statRow`
- Border: `borderBottomWidth: StyleSheet.hairlineWidth`, `borderBottomColor: colors.border` — matches existing `statRow`
- Label style: reuse `summaryStyles.statLabel` exactly (15px, semibold, `colors.secondary`)
- Value style: reuse `summaryStyles.statValue` exactly (15px, bold, `colors.primary`)
- Value format: `"[N] bpm"` — e.g. `"142 bpm"` — lowercase, space before unit, no leading zeros

Placement: Avg HR row first, Peak HR row second. Inserted after the "Exercises" row and before the "PRs" row (if present), so HR data groups with performance stats rather than workout structure stats.

### CalendarDayDetail — HR Second Line

HR stats rendered as a second `View` row below the existing `statsRow` in `SwipeableSessionCard`, rendered only when `session.avgHr !== null`:

```
[ Avg HR: [N] bpm ]  [ Peak HR: [N] bpm ]
```

- Layout: `flexDirection: 'row'`, `flexWrap: 'wrap'`, `gap: spacing.md` (12px) — matches `statsRow`
- Top margin: `marginTop: spacing.sm` (8px) — separates from main stats row visually
- Bottom margin: `marginBottom: spacing.base` (16px) — matches `statsRow.marginBottom`
- Each item: reuse `StatItem` component with `label="Avg HR"` / `label="Peak HR"` and value `"[N] bpm"`
- No border between the two HR stat rows and the main stats row — the second line stands on its own without a divider

Value format: consistent with WorkoutSummary — `"[N] bpm"` (e.g. `"142 bpm"`).

## Copywriting Contract

| Element | Copy |
|---------|------|
| Avg HR stat label | `Avg HR` |
| Peak HR stat label | `Peak HR` |
| Avg HR stat value format | `{N} bpm` (e.g. `142 bpm`) |
| Peak HR stat value format | `{N} bpm` (e.g. `178 bpm`) |
| Empty state (no HR data) | No visible change — HR rows are hidden entirely when `avg_hr` is null (D-08). No "No HR data" copy shown. |
| Error state | No HR-specific error state. Session end failure falls through to existing session error handling. |
| Loading state | No new loading state. WorkoutSummary is shown after flush completes synchronously on session end. CalendarDayDetail uses existing `ActivityIndicator` spinner with `colors.accent`. |
| Destructive confirmation | None — Phase 26 introduces no destructive actions. |

Notes:
- "bpm" is lowercase and follows the number with a space: `142 bpm`, not `142BPM` or `142bpm`.
- No degree symbol, no heart icon, no zone label on HR stats (D-02: zone display is Phase 27 scope).
- Partial data scenario (D-09): if the device disconnected mid-workout but samples were collected, avg/peak still render normally — no annotation like "(partial)" is shown.

## Interaction Contract

Phase 26 adds no new interactive elements. All new UI is display-only (read-only stat rows). No tap handlers, no gestures, no modals, no sheets.

The only user-facing interaction change is that the WorkoutSummary and CalendarDayDetail screens will conditionally contain additional rows — but those rows are non-interactive.

Accessibility: HR stat rows follow the same `View` + `Text` pattern as existing stat rows. No additional accessibility attributes required beyond what the existing `StatItem` component provides.
