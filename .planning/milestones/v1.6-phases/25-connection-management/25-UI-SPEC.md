## Interaction Contracts

### DeviceScanSheet

**Trigger:** Tap "Scan for Device" button in Settings HR card.

**Sheet presentation:**
- Slides up from bottom over the Settings screen (same as `ExercisePickerSheet`).
- Sheet height: 60% of screen height minimum, expandable to 85%.
- Backdrop: `#151718` at 50% opacity. Tapping backdrop dismisses sheet (same as `ExercisePickerSheet`).
- Sheet surface: `colors.surface` (`#1E2024`), borderRadius 20px at top corners.
- Drag handle: 4px × 32px pill, `colors.border`, centered, 12px from top of sheet.

**Scan lifecycle states:**

| State | UI |
|-------|-----|
| Scanning (active) | "Scanning..." heading with `ActivityIndicator` (size "small", color `colors.accent`) beside it. Timer label below: "Stops in Xs" counting down from 15, 13px bold, `colors.secondary`. Device rows appear as discovered. |
| Scan timed out (no devices found) | Empty state: heading "No heart rate monitors found." (17px bold, `colors.primary`), body "Make sure your device is on and in range." (15px regular, `colors.secondary`), "Scan Again" button (accent fill, full width, 44px min height). |
| Scan timed out (devices found) | Device list remains visible. "Scan Again" appears as secondary text button below the list (13px bold, `colors.accent`). |
| Connecting | Tapped device row shows inline `ActivityIndicator` replacing the chevron. Row is non-interactive. Other rows remain tappable. |
| Connection failed | Inline error beneath tapped device row: "Couldn't connect. Try again." (13px regular, `colors.danger`). Row returns to tappable state. |
| Connected | Sheet auto-closes. Settings screen shows updated paired device name. |

**Device list row (`DeviceListRow`):**
- Height: 52px (touch target 44px min, padding accounts for visual balance).
- Layout: [device name] — [signal bars] — [chevron ›]
- Device name: 17px bold, `colors.primary`, `numberOfLines={1}` with ellipsis.
- Signal bars (`SignalBars`): 3 bars, each 3px wide × varying height (8/12/16px), 2px gap between bars. Filled color `colors.accent`; unfilled color `colors.border`. RSSI thresholds: ≥−60 dBm = 3 bars, −61 to −80 = 2 bars, < −80 = 1 bar.
- Chevron: "›" text glyph, 17px bold, `colors.secondary`.
- Row separator: `StyleSheet.hairlineWidth` border, `colors.border`.
- Row background: `colors.surface`. On press: `colors.surfaceElevated` (activeOpacity 0.85).

**Age prompt (D-17 carried forward):**
Before scanning begins, if age not set in HRSettings:
- Inline prompt within the sheet (not a separate modal): show a single-line TextInput labeled "Enter your age to calculate HR zones" (13px regular, `colors.secondary`), numeric keyboard, with "Set Age" button (accent fill, 44px height). After valid age entry, scan starts automatically.

### HRConnectionIndicator (Workout Header)

**Placement:** In the `WorkoutScreen` header `View` (flexDirection: row), inserted between `volumeText` and `endButtonTouchable`. Aligned center vertically.

**Layout:** [colored dot] [space 4px] [state label]
- Dot: 8px × 8px, borderRadius 4px (circle).
- Label: 13px bold, color matches dot color.
- Entire indicator: rendered only when `pairedDeviceId` is set in HRSettings. Non-tappable — display only.

**BPM display (Phase 25 scope — "--" only):**
- When `pairedDeviceId` exists and `deviceState !== 'connected'`: display "--" in timerText position or as a secondary label beside the indicator. Use 15px regular, `colors.secondary`.
- Note: Live BPM number display is Phase 27 scope. Phase 25 only shows "--" or nothing.

**State transitions:**
- `connected`: dot `#8DC28A`, label "Connected"
- `reconnecting`: dot `#FACC15`, label "Reconnecting"
- `disconnected`: dot `#D9534F`, label "Disconnected"
- `scanning`: dot `#8DC28A` (dim: `colors.accentDim` background behind dot), label "Scanning"
- `connecting`: dot `#FACC15`, label "Connecting"

**Haptic on disconnect (D-13):** Single `HapticFeedback.trigger('impactMedium')` on transition from `connected` → `disconnected` or `connected` → `reconnecting`. Reuses existing `react-native-haptic-feedback` import in `WorkoutScreen`.

### Settings HR Card

**Primary focal point (no device paired):** "Scan for Device" accent-filled button — the only interactive element in the card when no device is paired, rendered at full card width and 44px height with `colors.accent` fill. This is the entry point to the entire connection management flow.

**Card structure** (inside `colors.surface` card, borderRadius 12px, padding `spacing.base`):

```
[Heart Rate Monitor]       ← cardTitle style (17px bold, colors.primary)
[Manage your paired device]← cardDescription style (13px regular, colors.secondary)

--- when no device paired ---
[Scan for Device]          ← accent button (44px min, full width) — PRIMARY FOCAL POINT

--- when device paired ---
[Device name]              ← 15px regular, colors.primary
[Connected · Reconnecting · Disconnected]  ← 13px bold, state color
[Scan for Device]          ← secondary action (accent text, not filled button)
[Unpair]                   ← 13px bold, colors.danger, right-aligned or full-width destructive
```

**Unpair interaction (D-11):**
- Tap "Unpair" → `Alert.alert('Unpair Device', 'Stop connecting to [device name] during workouts?', [{text:'Cancel'}, {text:'Unpair', style:'destructive'}])`.
- On confirm: calls `HRSettingsService.clearPairedDevice()`, disconnects BLE if connected, indicator disappears from workout header.
- Uses React Native `Alert.alert` — consistent with existing destructive patterns in the codebase (no custom confirmation modal needed).
