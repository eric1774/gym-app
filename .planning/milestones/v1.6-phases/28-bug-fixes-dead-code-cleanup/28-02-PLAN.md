# Phase 28 Plan 02: Dead Code Removal

## Objective

Remove dead types and exports that were identified in the v1.6 milestone audit: `HRSample` in `src/types/index.ts`, `getComputedMaxHR` in `HRSettingsService.ts`, and the `effectiveAge` fallback in `WorkoutScreen.tsx`.

## Context

Phase 27 delivered live BPM display with zone coloring. Post-delivery audit found three dead code items:

1. `HRSample` interface in `src/types/index.ts` — superseded by the local `HRSample` interface in `src/db/sessions.ts` which is what `HeartRateContext` actually imports.
2. `getComputedMaxHR` in `HRSettingsService.ts` — never called anywhere in production code; `SettingsScreen.tsx` imports it but never uses it.
3. `effectiveAge` fallback (`age ?? 35`) in `WorkoutScreen.tsx` bpmZone useMemo — dead because when `age === null && override !== null`, `computeMaxHR` ignores the age argument entirely. The fallback value of 35 is never reachable in a way that affects computation.

## Tasks

**type**: auto
**done**: `HRSample` interface is absent from `src/types/index.ts`; no TypeScript errors

Remove the `HRSample` interface (lines 285-292) from `src/types/index.ts`. The authoritative `HRSample` type lives in `src/db/sessions.ts` and is imported directly by `HeartRateContext.tsx`.

### Task 2: Remove getComputedMaxHR dead export from HRSettingsService.ts
**type**: auto

### Task 3: Remove effectiveAge fallback dead code from WorkoutScreen.tsx
**type**: auto

- Change `computeMaxHR(effectiveAge, override)` to `computeMaxHR(age!, override)` (using non-null assertion since the `age === null && override === null` guard ensures age is non-null when we reach this line without an override)
