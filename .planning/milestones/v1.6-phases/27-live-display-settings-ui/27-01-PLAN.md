**1. Create `src/utils/hrZones.ts`:**

Create a pure utility function for zone computation. This is the zone lookup logic consumed by HRLiveBpmDisplay.

```typescript
import { HR_ZONES, HRZoneInfo } from '../types';

export interface CurrentZoneResult {
  zone: HRZoneInfo | null;  // null when below Zone 1
  color: string;            // zone color or colors.secondary for below-zone
  label: string | null;     // "Zone N \u2014 Name" or null when no zone
}

export function getCurrentZone(bpm: number, maxHr: number): CurrentZoneResult {
  const percent = (bpm / maxHr) * 100;

  // Below Zone 1 (< 50% maxHR) per D-13
  if (percent < HR_ZONES[0].minPercent) {
    return { zone: null, color: '#8E9298', label: null };
  }

  // Find the matching zone (iterate from highest to lowest)
  for (let i = HR_ZONES.length - 1; i >= 0; i--) {
    if (percent >= HR_ZONES[i].minPercent) {
      const z = HR_ZONES[i];
      return {
        zone: z,
        color: z.color,
        label: `Zone ${z.zone} \u2014 ${z.name}`,
      };
    }
  }

  // Fallback (should not reach)
  return { zone: null, color: '#8E9298', label: null };
}
```

Key points:
- Uses em dash (\u2014) in zone label per copywriting contract: "Zone 3 \u2014 Aerobic"
- Above Zone 5 (>100% maxHR) stays as Zone 5 per D-14 because the loop finds Zone 5 first (its minPercent is 90%)
- Below Zone 1 returns gray (`#8E9298` = `colors.secondary`) with null label per D-13
- Pure function, no side effects, easily testable

**2. Create `src/components/HRLiveBpmDisplay.tsx`:**

Create the self-contained BPM display section. This component:
- Consumes `useHeartRate()` for `currentBpm` and `deviceState`
- Calls `getComputedMaxHR()` on mount to get max HR for zone computation
- Uses `getCurrentZone()` from the new utility for color/label
- Implements pulse animation using `Animated.sequence` + `Animated.timing`
- Renders `HRConnectionIndicator` inside itself (moved from header row)

Props: none (self-contained, reads from context).

Structure (top to bottom):
```
View (full-width)
  View (1px divider, backgroundColor: colors.border)
  View (paddingVertical: spacing.xl, alignItems: 'center')
    Animated.Text (BPM number — fontSize.xl=24px, weightBold, color = zone color or secondary)
    View (height: spacing.xs=4px gap)
    Text (zone label — fontSize.sm=13px, weightRegular, color: colors.secondary) — ONLY if label is not null
    View (height: spacing.sm=8px gap)
    HRConnectionIndicator (deviceState=deviceState, visible=true)
```

States:
- `deviceState !== 'connected'`: Show "- -" in `colors.secondary` (#8E9298), hide zone label per D-04
- `deviceState === 'connected'` and `currentBpm !== null`: Show BPM number with zone color, show zone label if in a zone
- `currentBpm` is null but connected: Show "- -" (waiting for first reading)

Pulse animation (per D-03 and UI-SPEC):
- Use `useRef(new Animated.Value(1))` for scale
- On each `currentBpm` change (only when value actually changes, tracked via `prevBpmRef`):
  - `Animated.sequence([ Animated.timing(scaleAnim, { toValue: 1.06, duration: 100, useNativeDriver: true }), Animated.timing(scaleAnim, { toValue: 1.0, duration: 100, useNativeDriver: true }) ]).start()`
- Do NOT pulse on initial mount, on "- -" state, or on re-renders without BPM change
- Use `useEffect` watching `currentBpm` with a `prevBpmRef` to detect actual changes

Zone color transition: instant snap per D-07 — just set the color directly from `getCurrentZone()` result, no interpolation.

Max HR loading:
- Call `getComputedMaxHR()` in a `useEffect` on mount, store result in state
- If maxHr is null (age not set), treat all BPM as below-zone (gray, no label)
- Re-call `getComputedMaxHR()` when component mounts (not on every render)
