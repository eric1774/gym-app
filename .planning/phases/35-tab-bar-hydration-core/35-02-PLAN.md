**handleSubmit:**
```typescript
const handleSubmit = useCallback(async () => {
  const parsed = parseFloat(amount);
  if (isNaN(parsed) || parsed <= 0) {
    setError('Please enter a valid amount');
    return;
  }
  setIsSubmitting(true);
  setError(null);
  try {
    await hydrationDb.logWater(Math.round(parsed));
    onSaved();
    handleClose();
  } catch (_err) {
    setError('Could not save — please try again');
    setIsSubmitting(false);
  }
}, [amount, onSaved, handleClose]);
```

**Data loading (useFocusEffect pattern, same as MacrosView):**
```typescript
const refreshData = useCallback(async () => {
  const [total, settings] = await Promise.all([
    hydrationDb.getTodayWaterTotal(),
    hydrationDb.getWaterGoal(),
  ]);
  setCurrentTotal(total);
  if (settings?.goalOz != null) {
    setGoalOz(settings.goalOz);
  }
  // else keep default 64 per D-09
}, []);

useFocusEffect(
  useCallback(() => {
    let cancelled = false;
    (async () => {
      await refreshData();  // Call shared refresh
      if (!cancelled) setIsLoading(false);
    })();
    return () => { cancelled = true; };
  }, [refreshData]),
);
```

NOTE: Must also call refreshData after every quick-add and after modal save, so `refreshData` needs to be a stable callback.

```typescript
import HapticFeedback from 'react-native-haptic-feedback';

const handleQuickAdd = useCallback(async (oz: number) => {

}, [refreshData]);
```

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-35-03 | Tampering | LogWaterModal input | mitigate | Validate: parseFloat must be > 0, NaN rejected, round to integer before DB write via `Math.round(parsed)` — prevents fractional injection or negative values |
| T-35-04 | Denial of Service | Quick-add rapid taps | accept | Each tap writes a small row to SQLite — rapid tapping produces many small inserts but SQLite handles this; no user-facing harm. No rate limiting needed for local-only app |
| T-35-05 | Information Disclosure | Error messages | accept | Error messages are generic ("Could not save — please try again") and do not expose DB internals or stack traces |
</threat_model>
