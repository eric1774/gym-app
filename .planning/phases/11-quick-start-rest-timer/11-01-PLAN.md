2. Add `getNextWorkoutDay()` function to `src/db/dashboard.ts`. Import `NextWorkoutInfo` from types. The query logic:

- Find the most recently used program (the program whose `program_day_id` was used in the most recent `workout_sessions` row). If no sessions exist with a `program_day_id`, fall back to the most recently created program.
- Only consider activated programs (where `start_date IS NOT NULL`).
- If no activated programs exist, return `null`.
- Call `getProgramWeekCompletion(programId)` to get the day completion status for the current week.
- Find the first day where `isCompletedThisWeek === false` — that is the next workout day.
- If ALL days are done for the current week, use the first day (the card always has something to show — per user decision "show the first day of the next week").
- Count exercises for that day: `SELECT COUNT(*) FROM program_day_exercises WHERE program_day_id = ?`.
- Return `NextWorkoutInfo` with programId, programName, dayId, dayName, exerciseCount.
- If exercise count is 0, still return the info (empty day is a valid state).

The function signature: `export async function getNextWorkoutDay(): Promise<NextWorkoutInfo | null>`

Implementation approach for finding the most recently used program:
```sql
SELECT DISTINCT p.id, p.name
FROM programs p
INNER JOIN program_days pd ON pd.program_id = p.id
INNER JOIN workout_sessions ws ON ws.program_day_id = pd.id
WHERE p.start_date IS NOT NULL
ORDER BY ws.started_at DESC
LIMIT 1
```
If no results, fall back to: `SELECT id, name FROM programs WHERE start_date IS NOT NULL ORDER BY created_at DESC LIMIT 1`
