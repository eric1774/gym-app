Create `src/screens/MealLibraryScreen.tsx` -- the full Meal Library screen.

**Imports needed:**
- React, useState, useCallback, useMemo, useRef from 'react'
- SectionList, View, Text, TouchableOpacity, Animated, PanResponder, StyleSheet, Alert from 'react-native'
- useFocusEffect from '@react-navigation/native'
- useSafeAreaInsets from 'react-native-safe-area-context'
- getLibraryMealsByType, deleteLibraryMeal, addMeal from '../db'
- LibraryMeal, MealType, MEAL_TYPES from '../types'
- AddLibraryMealModal from './AddLibraryMealModal'
- Theme tokens: colors, spacing, fontSize, weightBold, weightSemiBold, weightMedium

**Component structure:**

1. **State:**
   - `sections`: `Array<{ title: string; data: LibraryMeal[]; mealType: MealType }>` -- derived from getLibraryMealsByType, excluding empty types
   - `modalVisible`: boolean for AddLibraryMealModal
   - `toastMessage`: string | null for logging feedback
   - `isLoading`: boolean

2. **Data loading with useFocusEffect:**
   Call `getLibraryMealsByType()`, transform the Record into SectionList sections. For each MealType in MEAL_TYPES order, if the array is non-empty, create a section with `title` as the capitalized meal type name and `data` as the LibraryMeal array. Store in `sections` state. Use the cancelled-flag pattern from ProteinScreen.

3. **refreshData callback:**
   Same logic as useFocusEffect load, extracted into a reusable callback for use after add/delete operations.

4. **Header with '+' button:**
   Since headerShown is false in the ProteinStack navigator, build a custom header inside the component. Use useSafeAreaInsets for top padding. Layout:
   - Left: back arrow button (use `useNavigation().goBack()`) -- a TouchableOpacity with a '<' text or left-arrow character, minimum 44px touch target
   - Center: "Meal Library" title text
   - Right: '+' button (TouchableOpacity, minimum 44px, accent color, bold) that sets modalVisible to true
   Style the header with flexDirection 'row', alignItems 'center', paddingHorizontal spacing.base.

5. **SectionList:**
   - `sections` prop from state
   - `keyExtractor`: `(item) => String(item.id)`
   - `renderSectionHeader`: Render the section title (e.g. "Breakfast") with uppercase first letter, using fontSize.sm, color secondary, weightSemiBold, paddingHorizontal spacing.base, paddingTop spacing.lg, paddingBottom spacing.xs, backgroundColor colors.background
   - `renderItem`: Render a LibraryMealRow component (defined below)
   - `ListEmptyComponent`: When sections is empty and not loading, show centered "No saved meals yet" text (fontSize.base, color secondary) with a filled accent button "Add Meal" below (same style as ProteinScreen addMealButton) that opens the modal
   - `stickySectionHeadersEnabled={false}`

6. **LibraryMealRow (inner component, React.memo):**
   Adapt the MealListItem swipe-to-delete pattern. Props: `{ meal: LibraryMeal; onTap: (meal: LibraryMeal) => void; onDelete: (meal: LibraryMeal) => void }`.

   Layout (non-swiped state):
   - Row with flexDirection 'row', alignItems 'center', paddingHorizontal spacing.base, paddingVertical 14, minHeight 48
   - Left: `meal.name` text (fontSize.base, weightMedium, color primary, flex 1)
   - Right: `${meal.proteinGrams}g` text (fontSize.base, weightBold, color accent)
   - Bottom border: borderBottomWidth 1, borderBottomColor colors.border

   Swipe-to-delete: Same PanResponder pattern as MealListItem:
   - DELETE_THRESHOLD = -80
   - Animated.Value for translateX
   - onMoveShouldSetPanResponder: Math.abs(gs.dx) > 10 && Math.abs(gs.dx) > Math.abs(gs.dy)
   - Constrain to negative direction only (left swipe)
   - On release: snap to DELETE_THRESHOLD if past half, else snap back to 0
   - Delete button: absolute positioned red area on right, 80px wide, "Delete" text

   onPress of the row (not the delete area): calls onTap with the meal.

7. **One-tap logging handler:**
   ```typescript
   const handleTapMeal = useCallback(async (meal: LibraryMeal) => {
     try {
       await addMeal(meal.proteinGrams, meal.name, meal.mealType);
       setToastMessage(`${meal.name} ${meal.proteinGrams}g logged`);
       setTimeout(() => setToastMessage(null), 2000);
     } catch (_err) {
       Alert.alert('Error', 'Failed to log meal');
     }
   }, []);
   ```
   User stays on the library screen. No navigation. No confirmation dialog.

8. **Delete handler:**
   Use Alert.alert for delete confirmation (same pattern as ProteinScreen handleDelete), then call deleteLibraryMeal(id) and refreshData().

9. **Toast overlay:**
   Same pattern as ProteinScreen: absolute positioned View at bottom: 80, alignSelf 'center', surfaceElevated background, border, border radius 20. Conditionally rendered when toastMessage is not null.

10. **Modal integration:**
    Render `<AddLibraryMealModal visible={modalVisible} onClose={() => setModalVisible(false)} onSaved={() => { setModalVisible(false); refreshData(); }} />` at the bottom of the component return.

**Performance:** Use React.memo on LibraryMealRow, useCallback on handlers, useMemo where appropriate.

**Export:** Named export `export function MealLibraryScreen()`.

**Part A: Create AddLibraryMealModal**

Create `src/screens/AddLibraryMealModal.tsx` -- a simplified bottom-sheet modal for adding meals to the library. Follow the AddMealModal pattern but with only 3 fields (no date/time, no edit mode).

**Props interface:**
```typescript
interface AddLibraryMealModalProps {
  visible: boolean;
  onClose: () => void;
  onSaved: () => void;
}
```

**Imports:**
- React, useState from 'react'
- KeyboardAvoidingView, Modal, Platform, Pressable, ScrollView, StyleSheet, Text, TextInput, TouchableOpacity, View from 'react-native'
- MealTypePills from '../components/MealTypePills'
- addLibraryMeal from '../db'
- MealType from '../types'
- Theme tokens: colors, spacing, fontSize, weightBold, weightSemiBold

**Component:**
1. State: `name` (string), `proteinGrams` (string), `mealType` (MealType | null), `isSubmitting` (boolean), `error` (string | null)

2. handleClose: reset all state, call onClose

3. handleSubmit:
   - Guard: if isDisabled, return
   - Set isSubmitting true, clear error
   - Call `addLibraryMeal(name.trim(), parseFloat(proteinGrams), mealType!)`
   - On success: call onSaved(), then handleClose()
   - On error: set error message, set isSubmitting false

4. isDisabled: `name.trim() === '' || proteinGrams.trim() === '' || isNaN(parseFloat(proteinGrams)) || parseFloat(proteinGrams) <= 0 || mealType === null || isSubmitting`

5. Modal layout (same pattern as AddMealModal):
   - Modal with animationType="slide", transparent, onRequestClose={handleClose}
   - KeyboardAvoidingView with behavior="padding"
   - Pressable overlay (flex 1, rgba(0,0,0,0.6)) to dismiss on tap
   - View sheet (surface bg, top border radius 20, padding)
   - ScrollView with bounces={false}, keyboardShouldPersistTaps="handled"

6. Form fields:
   - Title: "Add to Library" (centered, bold, fontSize.lg)
   - "Meal Type" label + MealTypePills component
   - "Protein (grams)" label + TextInput with decimal-pad keyboard
   - "Name" label + TextInput with placeholder "e.g. Chicken breast"
   - Error text if present
   - Submit button: "Save to Library" (filled accent, same style as AddMealModal submit)
   - Cancel button below

7. Styles: Follow AddMealModal style patterns exactly (overlay, keyboardAvoid, sheet, title, label, fieldSpacing, input, submitButton, submitButtonDisabled, submitButtonText, cancelButton, cancelText, errorText).
