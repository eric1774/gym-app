Create `src/context/HeartRateContext.tsx` mirroring the TimerContext pattern (createContext + Provider + useHeartRate hook). The context manages a BLE state machine with 5 states from `DeviceConnectionState`.

**Context interface (HeartRateContextValue):**
```typescript
interface HeartRateContextValue {
  deviceState: DeviceConnectionState;        // current BLE state
  currentBpm: number | null;                 // latest HR reading (null when not connected)
  discoveredDevices: DiscoveredDevice[];      // devices found during scan
  pairedDeviceName: string | null;           // name of paired device for display
  startScan: () => Promise<void>;            // begin 15s BLE scan
  stopScan: () => void;                      // cancel active scan
  connectToDevice: (deviceId: string, deviceName: string) => Promise<void>; // pair + connect
  disconnect: () => Promise<void>;           // manual disconnect + clear paired
  attemptAutoReconnect: () => Promise<void>; // auto-reconnect to paired device
  scanTimeRemaining: number | null;          // countdown seconds (15 to 0), null when not scanning
}

interface DiscoveredDevice {
  id: string;
  name: string;
  rssi: number;
}
```

**State machine implementation details:**

1. **startScan():**
   - Call `requestBLEPermissions()` from BLEPermissions.ts first. If not 'granted', throw/return early.
   - Set `deviceState` to `'scanning'`.
   - Clear `discoveredDevices` array.
   - Start 15-second countdown timer (use `setInterval` every 1s, store in `scanTimerRef` useRef). Update `scanTimeRemaining` each tick. On expiry, call `stopScan()`.
   - Call `bleManager.startDeviceScan([HR_SERVICE_UUID], { allowDuplicates: false }, callback)`. In the callback: if device has a `name` (not null), add to `discoveredDevices` state (dedup by `device.id`). Extract `device.rssi` for signal strength.
   - Uses `HR_SERVICE_UUID` filter from BLEHeartRateService.ts to only show HR monitors (per Claude's discretion on scan filtering).

2. **stopScan():**
   - Call `bleManager.stopDeviceScan()`.
   - Clear the scan countdown interval (`scanTimerRef`).
   - Set `scanTimeRemaining` to null.
   - Set `deviceState` to `'disconnected'` (only if currently `'scanning'`).

3. **connectToDevice(deviceId, deviceName):**
   - Set `deviceState` to `'connecting'`.
   - Call `bleManager.stopDeviceScan()` (stop any active scan).
   - Call `bleManager.connectToDevice(deviceId, { timeout: 10000 })` (10s connection timeout per Claude's discretion).
   - On success: call `device.discoverAllServicesAndCharacteristics()`.
   - Monitor HR characteristic: call `device.monitorCharacteristicForService(HR_SERVICE_UUID, HR_MEASUREMENT_CHAR_UUID, callback)`. In callback, parse the HR measurement value from the characteristic `value` (base64-decode, byte 1 is BPM if flags byte bit 0 is 0, or bytes 1-2 little-endian uint16 if bit 0 is 1). Update `currentBpm`.
   - Call `setPairedDeviceId(deviceId)` to persist.
   - Store `deviceName` in state as `pairedDeviceName`.
   - Set `deviceState` to `'connected'`.
   - Register disconnect listener via `bleManager.onDeviceDisconnected(deviceId, callback)`. On disconnect callback: set `deviceState` to `'reconnecting'`, set `currentBpm` to null, start reconnect loop.

4. **Reconnect logic (per D-20: exponential backoff 1/2/4/8/16s, max 5 attempts):**
   - Store `reconnectAttemptRef = useRef(0)` and `reconnectTimerRef = useRef<NodeJS.Timeout | null>(null)`.
   - On disconnect detected: if `reconnectAttemptRef.current < 5`, schedule next attempt after `Math.pow(2, reconnectAttemptRef.current) * 1000` ms (gives 1s, 2s, 4s, 8s, 16s).
   - Each attempt: try `bleManager.connectToDevice(deviceId, { timeout: 10000 })`. On success: `reconnectAttemptRef.current = 0`, re-discover services, re-monitor characteristic, set `deviceState` to `'connected'`, re-register disconnect listener.
   - On failure: increment `reconnectAttemptRef.current`, schedule next attempt.
   - After 5 failures: set `deviceState` to `'disconnected'`, clear reconnect timer. No further attempts (per D-14).

5. **attemptAutoReconnect():**
   - Read `pairedDeviceId` from `getHRSettings()`.
   - If null, return (no paired device).
   - Set `deviceState` to `'connecting'`.
   - Attempt `bleManager.connectToDevice(pairedDeviceId, { timeout: 10000 })`.
   - On success: same flow as connectToDevice post-connect (discover services, monitor char, set connected, register disconnect listener). Read paired device name — `bleManager.connectToDevice` returns the Device object which has `.name`.
   - On failure: set `deviceState` to `'disconnected'` (silent timeout per D-10, no popup).

6. **disconnect():**
   - Call `clearPairedDevice()` from HRSettingsService.
   - Call `bleManager.cancelDeviceConnection(deviceId)` if connected.
   - Clear any reconnect timers.
   - Set `deviceState` to `'disconnected'`, `currentBpm` to null, `pairedDeviceName` to null.
   - Reset `reconnectAttemptRef.current` to 0.

7. **Cleanup on unmount:**
   - `useEffect` cleanup: stop scan, cancel device connection, clear all timers.

8. **HR measurement parsing helper (private function inside the file):**
```typescript
function parseHeartRate(base64Value: string): number {
  const bytes = Buffer.from(base64Value, 'base64');
  // BLE HR Measurement: byte 0 = flags, byte 1+ = HR value
  const flags = bytes[0];
  if (flags & 0x01) {
    // 16-bit HR value (little-endian)
    return bytes[1] | (bytes[2] << 8);
  }
  // 8-bit HR value
  return bytes[1];
}
```
Note: In React Native, use `atob` + manual byte conversion instead of Node `Buffer`. Implementation:
```typescript
function parseHeartRate(base64Value: string): number {
  const raw = atob(base64Value);
  const flags = raw.charCodeAt(0);
  if (flags & 0x01) {
    return raw.charCodeAt(1) | (raw.charCodeAt(2) << 8);
  }
  return raw.charCodeAt(1);
}
```
