**ProteinProgressBar.tsx** -- Shows protein progress with tap-to-edit goal:
- Props: `{ goal: number; current: number; onGoalChanged: (newGoal: number) => void }`
- Two modes controlled by local `isEditing` state:
  - **Display mode (default):** Wrapped in TouchableOpacity that sets isEditing=true on press.
    - Text showing `"{percentage}% -- {current}g / {goal}g"` (use em dash character \u2014)
    - Horizontal progress bar: outer View (progressTrack) with inner View (progressFill) using width percentage
    - Progress calculation: `const progress = goal > 0 ? Math.min(current / goal, 1) : 0;`
    - progressTrack: height 8px, backgroundColor colors.surfaceElevated, borderRadius 4, overflow hidden
    - progressFill: height 8px, backgroundColor colors.accent, borderRadius 4
  - **Edit mode:** Replaces progress bar with inline edit (per user decision -- NOT a modal)
    - TextInput pre-filled with current goal value, keyboardType="number-pad", autoFocus
    - Row with Save and Cancel buttons
    - Save: validate > 0, call `setProteinGoal(newValue)` from `'../db'`, call `onGoalChanged(newValue)`, set isEditing=false
    - Cancel: set isEditing=false (discard changes)
    - Wrap setProteinGoal in try/catch

Use theme tokens throughout. Follow AddExerciseModal styling patterns for inputs and buttons.
