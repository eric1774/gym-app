**MealListItem.tsx** -- Swipeable meal row with tap-to-edit and swipe-to-delete:
- Props: `{ meal: Meal; onEdit: (meal: Meal) => void; onDelete: (meal: Meal) => void }`
- Wrap in `React.memo` for FlatList performance (per mobile-design skill rule)
- Uses PanResponder + Animated for swipe-to-delete gesture (per research recommendation, NOT react-native-gesture-handler)

Swipe implementation:
- `const translateX = useRef(new Animated.Value(0)).current;`
- `const DELETE_THRESHOLD = -80;` (width of delete button area)
- PanResponder.create with:
  - `onMoveShouldSetPanResponder`: only capture when `Math.abs(gs.dx) > 10 && Math.abs(gs.dx) > Math.abs(gs.dy)` (prevents conflict with FlatList vertical scroll per pitfall 6)
  - `onPanResponderMove`: if gs.dx < 0 (left swipe only), `translateX.setValue(Math.max(gs.dx, DELETE_THRESHOLD))`
  - `onPanResponderRelease`: if gs.dx < DELETE_THRESHOLD / 2, snap open with `Animated.spring(translateX, { toValue: DELETE_THRESHOLD, useNativeDriver: true })`. Otherwise snap closed to 0.

Layout:
- Outer container View (relative position, overflow hidden)
- Delete button: positioned behind the row (absolute, right: 0, backgroundColor colors.danger, width: 80px, full height, centered "Delete" text)
  - onPress: calls `onDelete(meal)` (parent handles Alert.alert confirmation)
- Animated.View with translateX transform, spread panResponder.panHandlers:
  - TouchableOpacity wrapping meal content, onPress calls `onEdit(meal)`
  - Content row: meal type label (capitalize, colors.secondary, fontSize.xs), description if present (colors.primary, fontSize.base), protein grams on the right (weightBold, colors.accent, "{meal.proteinGrams}g")
  - Minimum row height 48px (touch target per mobile-design skill)
  - backgroundColor colors.surface, paddingHorizontal spacing.base, paddingVertical 14, borderBottomWidth 1, borderBottomColor colors.border

Use `useNativeDriver: true` on all Animated calls (per mobile-design skill anti-pattern list).
