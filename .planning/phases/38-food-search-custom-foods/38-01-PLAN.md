Row mappers:
- `rowToFood(row)`: Maps raw SQLite row to Food. Computes `caloriesPer100g` using `computeCalories(row.protein_per_100g, row.carbs_per_100g, row.fat_per_100g)`. Maps `is_custom` (integer 0/1) to boolean `isCustom`. Maps `fdc_id` to `fdcId` (nullable).
- `rowToFoodSearchResult(row)`: Calls `rowToFood(row)` then adds `usageCount: row.usage_count ?? 0`.

Functions:

`searchFoods(query: string): Promise<FoodSearchResult[]>`
- Trim and lowercase the query. If empty string after trim, return [].
- Split on whitespace into tokens. Filter out empty tokens.
- Build a single SQL query that LEFT JOINs foods with meal_foods to get usage counts, using AND-ed LIKE conditions for each token:
  ```sql
  SELECT f.*, COALESCE(mf_count.cnt, 0) as usage_count
  FROM foods f
  LEFT JOIN (
    SELECT food_id, COUNT(*) as cnt FROM meal_foods GROUP BY food_id
  ) mf_count ON mf_count.food_id = f.id
  WHERE f.search_text LIKE ? AND f.search_text LIKE ?
  ORDER BY usage_count DESC, f.name ASC
  LIMIT 20
  ```
  Each token becomes `%${token}%` as a parameter. All parameters are passed via parameterized query (per D-02 200ms debounce + D-04 token LIKE search + D-17 frequency boost + D-03 max 20 results, SRCH-01, SRCH-02, SRCH-04).

`getFrequentFoods(): Promise<FoodSearchResult[]>`
- Query foods that appear in meal_foods, ranked by count descending, limited to 10 (per D-06):
  ```sql
  SELECT f.*, COUNT(mf.id) as usage_count
  FROM foods f
  INNER JOIN meal_foods mf ON mf.food_id = f.id
  GROUP BY f.id
  ORDER BY usage_count DESC
  LIMIT 10
  ```
- Returns empty array if no meal_foods rows exist yet (per D-08).

`createCustomFood(name: string, proteinPer100g: number, carbsPer100g: number, fatPer100g: number): Promise<Food>`
- Validate name is non-empty (throw Error if blank).
- Validate macros are non-negative numbers.
- Insert into foods with `is_custom = 1`, `fdc_id = NULL`, `category = NULL`, `search_text = name.toLowerCase()` (per D-12, D-13, CUST-01).
- Use parameterized INSERT (SQL injection prevention).
- SELECT back the inserted row by insertId and return via `rowToFood()`.

`getFoodById(id: number): Promise<Food | null>`
- Simple SELECT by id, return null if no rows.

| T-38-04 | Denial of Service | searchFoods() | mitigate | LIMIT 20 on results prevents large result sets. Token splitting on whitespace is bounded by input length. 200ms debounce in UI layer prevents rapid-fire queries |
