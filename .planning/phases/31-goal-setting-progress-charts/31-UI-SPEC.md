## Spacing Scale

**Source:** `src/theme/spacing.ts` — project uses a non-standard scale that includes 12px and 20px tokens. This phase is constrained to use only values from the standard 8-point set (4, 8, 16, 24, 32, 48, 64px). Any reference to `spacing.md` (12px) or `spacing.lg` (20px) in existing code must not be introduced in new Phase 31 components.

| Token | Value | Usage in this phase |
|-------|-------|---------------------|
| `spacing.xs` | 4px | Icon gaps, label-to-bar gap, inline padding, error text `marginTop` |
| `spacing.sm` | 8px | Tight internal gaps, button row gap, card header vertical padding, inline edit input vertical padding, input field gap, tab row `gap`, legend row `paddingBottom`, filter pill gap |
| `spacing.base` | 16px | Card horizontal/vertical padding, screen horizontal padding, legend row `paddingHorizontal`, legend row `paddingTop`, subtitle `marginBottom` |
| `spacing.xl` | 24px | Card padding for MacroGoalSetupForm, section separators |
| `spacing.xxl` | 32px | Major section breaks between components |
| `spacing.xxxl` | 48px | MacroGoalSetupForm top margin (first-time empty state) |

**Exceptions:**
- Progress track bar height: 8px (compact multi-bar layout; 8px is a standard-set multiple of 4, replacing the prior 10px non-conforming value)
- Each bar row minimum touch target: 44px height (to allow inline edit trigger via tap)

**Non-standard token mapping (do not use in this phase):**

| Project token | Project value | Replacement for this phase |
|--------------|--------------|---------------------------|
| `spacing.md` | 12px | Map to `spacing.sm` (8px) for tight contexts (card header vertical padding, input vertical padding) or `spacing.base` (16px) for section-level gaps |
| `spacing.lg` | 20px | Map to `spacing.base` (16px) for between-section gaps |

### Accent Reserved For

The three macro colors (`#8DC28A`, `#5B9BF0`, `#E8845C`) are reserved for:
1. Progress bar fill for the corresponding macro row
2. The single-letter label (P, C, F) to the left of each bar
3. Active tab text label in the MacroChart tab selector
4. Chart `LineChart` dataset `color` for the active macro's line
5. The active inline edit input `borderColor` (that macro's color only)

`colors.accent` (#8DC28A) is additionally reserved for:
6. The "Set Goals" primary action button background in MacroGoalSetupForm
7. The "Save Goal" button background in per-bar inline edit mode

No other elements may use any of these four accent values.

### Goal Reference Line

The dashed goal reference line in MacroChart uses `colors.secondary` (#8E9298), `strokeWidth: 1`, dashed via `strokeDasharray`. This matches the existing `ProteinChart` goal line pattern.

## Component Inventory

### New Components This Phase

#### `MacroProgressCard` (new — replaces `ProteinProgressBar`)

Single card containing three macro progress rows plus a calorie total header.

**Card structure:**
```
[Section header: "DAILY MACROS" — uppercase, fontSize.sm, weightBold, colors.secondary, letterSpacing: 1.2]
[Card: colors.surface, borderRadius: 14, borderWidth: 1, borderColor: colors.border]
  [Header row: "DAILY MACROS" left | "X,XXX cal" right]
  [1px divider: colors.border]
  [Protein bar row — or inline edit row — or unset placeholder row]
  [1px divider: colors.border]
  [Carbs bar row — or inline edit row — or unset placeholder row]
  [1px divider: colors.border]
  [Fat bar row — or inline edit row — or unset placeholder row]
```

**Card header row:**
- Left: "DAILY MACROS" — `fontSize.sm`, `weightBold`, `colors.secondary`, `letterSpacing: 1.2`
- Right: computed calorie total — `fontSize.md`, `weightBold`, `colors.primary` — e.g. "1,840 cal"
- Padding: `spacing.base` horizontal, `spacing.sm` vertical
- Calorie total source: `computeCalories(todayTotals.protein, todayTotals.carbs, todayTotals.fat)` from `src/utils/macros.ts`

**Active bar row (goal set):**
- Height: minimum 44px (touch target)
- Padding: `spacing.base` horizontal, `spacing.sm` vertical
- Left side: macro letter label (P/C/F) in `MACRO_COLORS[macroType]`, `fontSize.sm`, `weightBold`, fixed width 16px
- Center: progress track (full flex-1 width), height 8px, background `#33373D`, `borderRadius: 4`
  - Fill: `MACRO_COLORS[macroType]`, `borderRadius: 4`, width as `Math.max(percentage, 8)%`
- Right side: percentage text — `fontSize.sm`, `weightBold`, `MACRO_COLORS[macroType]` — e.g. "75%", fixed width 36px, right-aligned
- Below track: gram text — `fontSize.base`, `weightRegular`, `colors.secondary` — e.g. "150g / 200g", `paddingTop: spacing.xs`
- Entire row is `TouchableOpacity`, `activeOpacity: 0.7`, tapping triggers inline edit for that macro

**Unset placeholder row (goal is NULL):**
- Height: minimum 44px (same as active bar row — maintains vertical rhythm)
- Padding: `spacing.base` horizontal, `spacing.sm` vertical
- Content: macro letter (P/C/F) in `colors.secondary` + em dash line + "Tap to set goal"
- Copy: `C ───  Tap to set goal` — letter in `colors.secondary`, remainder in `colors.secondary`, `fontSize.base`, `weightRegular`
- The em dash sequence is decorative spacing only — not an actual series of em dashes. Use a View with a 1px height line `flex: 1` for the line, not characters.
- Entire row is `TouchableOpacity`, tapping opens inline edit for that macro

**Inline edit row (state: isEditing for that macro):**
- Replaces the bar row for the edited macro only; other bars remain visible
- Input: `backgroundColor: colors.surfaceElevated`, `borderRadius: 10`, `borderWidth: 2`, `borderColor: MACRO_COLORS[macroType]`, `paddingVertical: spacing.sm`, `paddingHorizontal: spacing.base`, `fontSize.base`, `colors.primary`, `keyboardType="number-pad"`, `maxLength={5}`, `autoFocus`, `returnKeyType="done"`
- Button row below input: "Save Goal" + "Discard" side by side, `gap: spacing.sm`
  - Save Goal: `flex: 1`, `backgroundColor: colors.accent`, `borderRadius: 12`, `paddingVertical: spacing.sm`, label `colors.onAccent`, `fontSize.sm`, `weightBold`
  - Discard: `flex: 1`, `backgroundColor: colors.surfaceElevated`, `borderRadius: 12`, `paddingVertical: spacing.sm`, label `colors.secondary`, `fontSize.sm`, `weightBold`
- Error text: `fontSize.sm`, `colors.danger`, `marginTop: spacing.xs`

**Filter pills (below chart card):**

## Interaction Contracts

### Bar Tap — Inline Edit Trigger

- **Trigger:** Tap on any active bar row or unset placeholder row in MacroProgressCard
- **Effect:** That row swaps to inline edit mode (TextInput + "Save Goal"/"Discard"). Other two bars remain in display mode. Only one macro edits at a time.
- **Input pre-fill:** If goal was set, pre-fill with current goal value as string. If unset (NULL), pre-fill empty.
- **Save Goal:** Calls `macrosDb.setMacroGoals({ [macroType]: value })` with partial update. On success, row reverts to display mode with new value. On error, show inline error text without closing edit.
- **Discard:** Reverts to display mode with no change. No confirmation needed — user tapped "Discard" explicitly.
- **Input border color:** `MACRO_COLORS[macroType]` (2px width) — protein = mint, carbs = blue, fat = coral.
- **Keyboard dismissal:** `returnKeyType="done"` triggers save attempt.

### Tab Switch — Chart Macro Selection

- **Trigger:** Tap Protein, Carbs, or Fat tab in MacroChart
- **Effect:** Active tab visual updates instantly. `LineChart` datasets swap to selected macro's column from existing `MacroChartPoint[]` data. No DB re-fetch.
- **Goal reference line:** Appears only when selected macro's goal in MacroSettings is non-NULL. If NULL, datasets array contains only the intake dataset.
- **No animation on tab switch** — instant re-render.

### Time Range Filter — Chart Range Selection

- **Trigger:** Tap 1W, 1M, 3M, or All pill below chart
- **Effect:** Calls `macrosDb.getDailyMacroTotals(startDate, endDate)`. On resolved data, chart updates. `useFocusEffect` fires on screen focus and on range/refreshKey change.
- **Loading state:** No explicit spinner — existing `ProteinChart` pattern (chart just shows prior data until new data arrives).

### Goal Form Save

- **Trigger:** Tap "Set Goals" button in MacroGoalSetupForm
- **Validation:** Protein must be a positive integer > 0. Carbs and Fat default to 0 if empty.
- **Save:** Calls `macrosDb.setMacroGoals({ protein: p, carbs: c, fat: f })`. On success, form is replaced by MacroProgressCard.
- **Live calorie estimate:** Updates on every keystroke via controlled component state. Shows `~ {computeCalories(p, c, f)} calories/day`.
- **Button disabled state:** `opacity: 0.5` when protein field is empty or non-positive.

### Bar Fill Animation

- **Policy:** No fill animation. Progress bars render at their computed width instantly.
- **Rationale:** Three bars rendering simultaneously — simultaneous animations would add visual noise. Snap render is consistent with the app's speed-first UX priority.
