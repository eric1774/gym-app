**Props interface:**
```typescript
interface MacroProgressCardProps {
  goals: MacroSettings;        // From macrosDb.getMacroGoals()

}
```

**Imports:**
- `React, { useState }` from 'react'

- `{ spacing }` from '../theme/spacing'
- `{ fontSize, weightBold, weightRegular }` from '../theme/typography'

**Component structure (per D-01, UI-SPEC MacroProgressCard section):**

1. **Outer wrapper:** `paddingHorizontal: spacing.base`, `marginTop: spacing.base` (same as ProteinProgressBar wrapper)

2. **Card container:** `backgroundColor: colors.surface`, `borderRadius: 14`, `borderWidth: 1`, `borderColor: colors.border`, `overflow: 'hidden'`

3. **Header row (per D-10):**
   - Left: "DAILY MACROS" — `fontSize.sm` (13px), `weightBold`, `colors.secondary`, `letterSpacing: 1.2`
   - Right: calorie total — `fontSize.md` (17px), `weightBold`, `colors.primary`, formatted as `{N} cal` with comma separator (e.g. "1,840 cal")
   - Calorie total computed via `computeCalories(todayTotals.protein, todayTotals.carbs, todayTotals.fat)`
   - Padding: `spacing.base` horizontal, `spacing.sm` vertical
   - Format calorie number with `toLocaleString()` for comma separator

4. **Three bar rows** — render for each macro type in order: `['protein', 'carbs', 'fat']`
   - 1px divider (`height: 1, backgroundColor: colors.border`) between header and first row, and between each row
   - Map macro type to goal value: protein -> `goals.proteinGoal`, carbs -> `goals.carbGoal`, fat -> `goals.fatGoal`
   - Map macro type to label letter: protein -> "P", carbs -> "C", fat -> "F"
   - Map macro type to current value: `todayTotals[macroType]`

5. **State:** `editingMacro: MacroType | null` — tracks which macro is being inline-edited (only one at a time, per D-05)

**Active bar row (when goal is set, i.e., goal value is not null) — per UI-SPEC:**
- Height: minimum 44px touch target
- Padding: `spacing.base` horizontal, `spacing.sm` vertical
- Layout: `flexDirection: 'row'`, `alignItems: 'center'`
- Left: macro letter (P/C/F) in `MACRO_COLORS[macroType]`, `fontSize.sm`, `weightBold`, `width: 16`
- Center: progress track container (`flex: 1`, `marginHorizontal: spacing.sm`)
  - Track: `height: 8`, `backgroundColor: '#33373D'`, `borderRadius: 4`
  - Fill: `MACRO_COLORS[macroType]`, `borderRadius: 4`, width = `Math.min(Math.max((current/goal)*100, 8), 100)%` — min 8% visibility, capped at 100%
- Right: percentage text — `fontSize.sm`, `weightBold`, `MACRO_COLORS[macroType]`, `width: 36`, `textAlign: 'right'`
- Below track row: gram text — `fontSize.base` (15px), `weightRegular`, `colors.secondary`, text: `"{current}g / {goal}g"`, `paddingTop: spacing.xs`
- Entire row wrapped in `TouchableOpacity` with `activeOpacity: 0.7`, `onPress` triggers inline edit for that macro

**Unset placeholder row (when goal is null) — per D-02:**
- Height: minimum 44px (same as active row)
- Padding: `spacing.base` horizontal, `spacing.sm` vertical
- Layout: `flexDirection: 'row'`, `alignItems: 'center'`
- Macro letter in `colors.secondary`, `fontSize.sm`, `weightBold`, `width: 16`
- Decorative line: `View` with `flex: 1`, `height: 1`, `backgroundColor: colors.border`, `marginHorizontal: spacing.sm`
- Text: "Tap to set goal" — `colors.secondary`, `fontSize.base`, `weightRegular`
- Entire row is `TouchableOpacity`, tapping opens inline edit for that macro (per D-06)

**Inline edit row (when `editingMacro === macroType`) — per D-05:**
- State: `editValue: string` (pre-filled with current goal value as string if set, empty if null), `editError: string | null`
- Input: `backgroundColor: colors.surfaceElevated`, `borderRadius: 10`, `borderWidth: 2`, `borderColor: MACRO_COLORS[macroType]`, `paddingVertical: spacing.sm`, `paddingHorizontal: spacing.base`, `fontSize.base`, `colors.primary`, `keyboardType: "number-pad"`, `maxLength: 5`, `autoFocus: true`, `returnKeyType: "done"`, `onSubmitEditing` triggers save
- Button row: `flexDirection: 'row'`, `gap: spacing.sm`, `marginTop: spacing.sm`
  - "Save Goal": `flex: 1`, `backgroundColor: colors.accent`, `borderRadius: 12`, `paddingVertical: spacing.sm`, label `colors.onAccent`, `fontSize.sm`, `weightBold`
  - "Discard": `flex: 1`, `backgroundColor: colors.surfaceElevated`, `borderRadius: 12`, `paddingVertical: spacing.sm`, label `colors.secondary`, `fontSize.sm`, `weightBold`
- Error text: `fontSize.sm`, `colors.danger`, `marginTop: spacing.xs`
- Save logic: parse `editValue` as integer, validate > 0, call `macrosDb.setMacroGoals({ [macroType]: parsedValue })`, call `onGoalChanged()` on success, show error on failure
- Save error text: "Please enter a number greater than 0" for invalid input, "Failed to update goal. Please try again." for DB error
- Discard: set `editingMacro` to null, clear error

**Helper function** `formatCalories(n: number): string` — returns `n.toLocaleString()` for comma-separated display.

Create `src/components/MacroGoalSetupForm.tsx` — a 3-input macro goal setup form shown to first-time users when no macro_settings row exists. This replaces `GoalSetupForm.tsx` (per D-03).

**Props interface:**
```typescript
interface MacroGoalSetupFormProps {
  onGoalSet: () => void;   // Callback after goals are saved successfully
}
```

**Imports:**
- `React, { useState, useRef }` from 'react'
- `{ StyleSheet, Text, TextInput, TouchableOpacity, View }` from 'react-native'
- `{ macrosDb }` from '../db'
- `{ computeCalories }` from '../utils/macros'
- `{ colors }` from '../theme/colors'
- `{ spacing }` from '../theme/spacing'
- `{ fontSize, weightBold, weightRegular }` from '../theme/typography'

**Component structure (per D-03, D-04, D-11, UI-SPEC MacroGoalSetupForm section):**

Card wrapper: `backgroundColor: colors.surface`, `borderRadius: 16`, `padding: spacing.xl` (24px), `marginHorizontal: spacing.base`, `marginTop: spacing.xxxl` (48px), `alignItems: 'center'`

1. **Title:** "Set your daily goals" — `fontSize.lg` (20px), `weightBold`, `colors.primary`, `marginBottom: spacing.sm`, `textAlign: 'center'`

2. **Subtitle:** "Set daily targets for your macros" — `fontSize.sm` (13px), `colors.secondary`, `textAlign: 'center'`, `marginBottom: spacing.base`

3. **Three input fields** stacked with `spacing.sm` (8px) gap between them:
   - Each has a label above: "Protein (g) *", "Carbs (g)", "Fat (g)" — `fontSize.sm`, `weightBold`, `colors.secondary`, `marginBottom: spacing.xs`
   - Input: `backgroundColor: colors.surfaceElevated`, `borderRadius: 8`, `paddingVertical: spacing.sm`, `paddingHorizontal: spacing.base`, `fontSize.base`, `colors.primary`, `width: '100%'`, `textAlign: 'center'`, `keyboardType: "number-pad"`, `maxLength: 5`
   - Protein: `returnKeyType: "next"` — advances focus to Carbs input via ref
   - Carbs: `returnKeyType: "next"` — advances focus to Fat input via ref
   - Fat: `returnKeyType: "done"`
   - Use `useRef<TextInput>(null)` for carbsRef and fatRef to manage focus chaining via `onSubmitEditing`
   - Protein is required (asterisk in label). Carbs and Fat are optional.

4. **"* Required" note:** below Fat input — `fontSize.sm`, `colors.secondary`, `marginTop: spacing.xs`, `textAlign: 'center'`

5. **Live calorie estimate (per D-11):**
   - Below inputs: `marginTop: spacing.base`
   - Text: `~ {N} calories/day` — `fontSize.md` (17px), `weightBold`, `colors.primary`, `textAlign: 'center'`
   - N = `computeCalories(proteinVal || 0, carbsVal || 0, fatVal || 0)` where each val is `parseInt(inputString, 10)` or 0 if NaN
   - Format N with `toLocaleString()` for comma separator
   - When all inputs empty: shows "~ 0 calories/day"

6. **Error text:** `fontSize.sm`, `colors.danger`, `marginTop: spacing.xs`, `textAlign: 'center'`
   - "Please enter a protein goal greater than 0" for invalid protein
   - "Failed to set goals. Please try again." for DB error

7. **"Set Goals" button:**
   - `backgroundColor: colors.accent`, `borderRadius: 12`, `paddingVertical: spacing.base`, `width: '100%'`, `marginTop: spacing.base`, `alignItems: 'center'`
   - Label: "Set Goals" — `color: colors.background` (#151718), `fontSize.base`, `weightBold`
   - Disabled state: `opacity: 0.5` when protein input is empty or non-numeric positive integer
   - Enabled: protein has valid positive integer (carbs/fat default to 0 if empty)

**State:**
- `proteinValue: string`, `carbsValue: string`, `fatValue: string` — controlled text inputs
- `isSubmitting: boolean` — disable button during save
- `error: string | null`

**Save logic:**
- Parse protein as integer, validate > 0
- Parse carbs and fat as integers, default 0 if empty/NaN
- Call `macrosDb.setMacroGoals({ protein: p, carbs: c || undefined, fat: f || undefined })` — only send carbs/fat if > 0, otherwise let them remain null in DB
- On success: call `onGoalSet()`
- On error: set error message

Export as named export: `export function MacroGoalSetupForm(...)`.
