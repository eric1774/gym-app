**Key insight from Phase 15-02:** BackgroundTimer mock delegates to global.setInterval/clearInterval, so `jest.useFakeTimers()` controls the timer. Each `jest.advanceTimersByTime(1000)` triggers one tick of the countdown.

**Setup:**
```typescript
import React from 'react';
import { Text, View } from 'react-native';
import { render, act, waitFor } from '@testing-library/react-native';
import { TimerProvider, useTimer } from '../TimerContext';
import HapticFeedback from 'react-native-haptic-feedback';
```

Sound instances are created at module scope in TimerContext.tsx (countdownBeep, countdownDone). Since __mocks__/react-native-sound.js auto-mocks the constructor, the play/stop methods are available as jest.fn() stubs on any Sound instance. To spy on them, get references after importing:
```typescript
import Sound from 'react-native-sound';
```
Since Sound is mocked, instances created at module load time will have `.play` and `.stop` as jest.fn(). Access them by getting the mock instances from Sound constructor calls:
```typescript
// After TimerContext loads, Sound was called twice (countdownBeep, countdownDone)
// We can spy on the specific instance methods via Sound mock
```

Actually, the simplest approach: since the Sound mock returns objects with play/stop as jest.fn(), we can get the instances from the Sound constructor mock calls. But this is fragile. Better approach: just verify HapticFeedback.trigger was called (which is the directly observable side effect), and optionally check Sound instance method calls.

**TestConsumer pattern:**
```typescript
function TimerTestConsumer({ onCtx }: { onCtx?: (ctx: any) => void }) {
  const ctx = useTimer();
  React.useEffect(() => { onCtx?.(ctx); });
  return (
    <View>
      <Text testID="remaining">{String(ctx.remainingSeconds)}</Text>
      <Text testID="total">{String(ctx.totalSeconds)}</Text>
      <Text testID="running">{String(ctx.isRunning)}</Text>
    </View>
  );
}
```

Use `jest.useFakeTimers()` in beforeEach and `jest.useRealTimers()` in afterEach.

**Tests (7 total):**

1. `starts in idle state`: render `<TimerProvider><TimerTestConsumer /></TimerProvider>`, assert remaining='null', total='null', running='false'.

2. `startTimer sets running and remaining`: get ctxRef via onCtx callback. `await act(async () => { ctxRef.startTimer(10); })`. Assert remaining='10', total='10', running='true'.

3. `countdown decrements each second`: startTimer(5). Then `act(() => { jest.advanceTimersByTime(1000); })`. Assert remaining='4'. Advance another 1000ms. Assert remaining='3'.

4. `plays beep and haptic at 3, 2, 1 seconds`: startTimer(5). Advance 2 seconds (remaining=3). Check `HapticFeedback.trigger` was called with `'impactLight'`. Clear mock. Advance 1 second (remaining=2). Check haptic again. Advance 1 second (remaining=1). Check haptic again.

5. `completes with done sound and double haptic at 0`: startTimer(3). Advance 3 seconds total (remaining reaches 0). Assert `HapticFeedback.trigger` was called with `'notificationSuccess'`. Then advance 400ms (the setTimeout for second haptic pulse). Assert `HapticFeedback.trigger` called with `'notificationSuccess'` again (total 2 calls of notificationSuccess).

6. `stopTimer clears interval and resets state`: startTimer(10). Advance 3 seconds (remaining=7). Call `await act(async () => { ctxRef.stopTimer(); })`. Assert remaining='null', total='null', running='false'.

7. `cleans up interval on unmount`: startTimer(10). Unmount the component via `unmount()`. Advance timers. No errors thrown. This verifies the useEffect cleanup runs clearExistingInterval.

**Important notes on act():**
- Wrap all timer advances in `act(() => { ... })` to flush React state updates
- The BackgroundTimer.setInterval callback is async (it calls showTimerNotification which is async), so may need `await act(async () => { jest.advanceTimersByTime(1000); })` to let promises resolve
- Use `jest.runAllTimers()` sparingly -- prefer `jest.advanceTimersByTime()` for precise control
